### Vulnérabilités critiques dans Citrix NetScaler ADC et Gateway (Citrix Bleed 2)
Une vulnérabilité critique, surnommée "Citrix Bleed 2" (CVE-2025-5777), a été découverte dans NetScaler ADC et Gateway, similaire à l'originale Citrix Bleed (CVE-2023-4966). Il s'agit d'une vulnérabilité de lecture de mémoire hors limites qui permet aux attaquants non authentifiés d'accéder à des portions de mémoire normalement inaccessibles. Cela pourrait permettre le vol de jetons de session, d'identifiants et d'autres données sensibles, permettant aux attaquants de détourner les sessions utilisateur et de contourner l'authentification multi-facteurs (MFA). ReliaQuest estime avec une confiance moyenne que cette vulnérabilité est activement exploitée. Des vulnérabilités multiples dans les produits Citrix ont été découvertes, dont la plus grave pourrait entraîner la divulgation de données sensibles, notamment des jetons de session authentifiés, permettant un accès ultérieur à l'appliance ou aux systèmes.
* Publication date : 2025/06/27
* 🌐 Source : hxxps[:]//www[.]bleepingcomputer[.]com/news/security/citrix-bleed-2-flaw-now-believed-to-be-exploited-in-attacks/, hxxps[:]//www[.]cisecurity[.]org/advisory/multiple-vulnerabilities-in-citrix-products-could-allow-for-disclosure-of-sensitive-data_2025-060
* 💥 CVE : CVE-2025-5777, CVE-2023-4966
* 📦 CVE Impacted Product : Citrix NetScaler ADC, Citrix NetScaler Gateway, Produits Citrix
* 🛡️ Security recommandations : Examiner les systèmes Citrix pour des signes d'exploitation et appliquer les correctifs dès que disponibles. Mettre en place des mesures de détection et de réponse pour les tentatives de vol de jetons de session ou de contournement MFA.

### Vulnérabilités critiques RCE dans Cisco ISE et ISE-PIC (CVE-2025-20281, CVE-2025-20282)
Des failles critiques d'exécution de code à distance (RCE) ont été découvertes dans Cisco Identity Services Engine (ISE) et ISE Passive Identity Connector (ISE-PIC).
*   **CVE-2025-20281** (CVSS 9.8) : Causée par une validation insuffisante des entrées utilisateur dans une API exposée publiquement. Permet à des attaquants distants et non authentifiés d'exécuter des commandes arbitraires du système d'exploitation avec les privilèges root.
*   **CVE-2025-20282** (CVSS 10.0) : Causée par une validation de fichier inadéquate dans une API interne. Permet aux adversaires non authentifiés de télécharger et d'exécuter des fichiers arbitraires dans des répertoires système protégés, menant également à un accès de niveau root.
Ces vulnérabilités ne nécessitent aucune authentification ni interaction utilisateur, augmentant considérablement le risque pour les entreprises, les agences gouvernementales et les infrastructures de fournisseurs de services. Bien qu'aucune exploitation active n'ait été détectée, une action immédiate est fortement recommandée.
* Publication date : 2025/06/27
* 🌐 Source : hxxps[:]//socprime[.]com/blog/cve-2025-20281-and-cve-2025-20282-vulnerabilities/
* 💥 CVE : CVE-2025-20281, CVE-2025-20282
* 📦 CVE Impacted Product : Cisco Identity Services Engine (ISE), Cisco ISE Passive Identity Connector (ISE-PIC) versions 3.3 et 3.4 (CVE-2025-20281), ISE/ISE-PIC 3.4 (CVE-2025-20282)
* 📈 CVSS : 9.8 (CVE-2025-20281), 10.0 (CVE-2025-20282)
* 🛡️ Security recommandations : Appliquer les correctifs appropriés : ISE/ISE-PIC 3.3 Patch 6 et 3.4 Patch 2. Aucune solution de contournement n'est disponible.

*   **Game Users Share Buttons Plugin (CVE-2025-6755)** : Vulnérable à la suppression de fichiers arbitraires (jusqu'à la version 1.3.0) via la fonction `ajaxDeleteTheme()` en raison d'une validation de chemin de fichier insuffisante. Des attaquants de niveau abonné peuvent ajouter des chemins de fichiers arbitraires, pouvant conduire à une exécution de code à distance.
    *   📈 CVSS : 8.8 (ÉLEVÉE)
*   **PT Project Notebooks Plugin (CVE-2025-5304)** : Vulnérable à l'élévation de privilèges (versions 1.0.0 à 1.1.3) en raison d'une autorisation manquante dans la fonction `wpnb_pto_new_users_add()`. Permet aux attaquants non authentifiés d'élever leurs privilèges au rang d'administrateur.
    *   📈 CVSS : 9.8 (CRITIQUE)
*   **BeeTeam368 Extensions Plugin (CVE-2025-6381)** et **BeeTeam368 Extensions Pro Plugin (CVE-2025-6379)** : Vulnérables à la traversée de répertoire (jusqu'à la version 2.3.4) via les fonctions `handle_remove_temp_file()` et `handle_live_fn()` respectivement. Des attaquants authentifiés (niveau abonné ou supérieur) peuvent manipuler des fichiers en dehors du répertoire prévu, potentiellement pour supprimer `wp-config.php` et prendre le contrôle du site.
    *   📈 CVSS : 8.8 (ÉLEVÉE)
* Publication date : 2025/06/28

### Protection contre les attaques VMware vCenter et CVE-2023-34048
Une vulnérabilité critique d'exécution de code à distance (RCE) dans VMware vCenter Server (CVE-2023-34048), corrigée en octobre 2023, a été activement exploitée en janvier 2024. Cette faille permettait aux attaquants de prendre le contrôle total de l'infrastructure virtuelle d'une organisation sans authentification. Les méthodes d'attaque incluent l'établissement de la persistance en téléchargeant/téléchargeant des fichiers ISO, créant des machines virtuelles (VM) malveillantes ("VirtualGHOST") qui contournent les méthodes de création standard et les capacités de journalisation de VMware. Les adversaires peuvent également cibler les contrôleurs de domaine (DC) virtualisés pour l'accès aux identifiants (par exemple, en montant le disque virtuel du DC et en vidant la base de données NTDS.dit).
* Publication date : 2025/06/27
* 🌐 Source : hxxps[:]//www[.]crowdstrike[.]com/en-us/blog/falcon-next-gen-siem-protects-against-vmware-vcenter-attacks/
* 💥 CVE : CVE-2023-34048
* 📦 CVE Impacted Product : VMware vCenter Server
* 🛡️ Security recommandations : Mettre à jour VMware vCenter Server vers les versions corrigées. Ingesting les journaux vCenter dans un SIEM (par exemple, Falcon Next-Gen SIEM) et utiliser des règles de détection pour identifier les activités suspectes, telles que le téléchargement d'ISO, la création de VM non autorisées et les tentatives d'accès aux disques virtuels des contrôleurs de domaine. Renforcer les protections ESXi.

### Faille critique dans Open VSX Registry permettant des attaques sur la chaîne d'approvisionnement des développeurs
Des chercheurs en cybersécurité ont découvert une vulnérabilité critique dans le registre Open VSX (open-vsx[.]org), un hub d'extensions pour VS Code, qui aurait pu permettre aux attaquants de prendre le contrôle de l'ensemble du marché des extensions. Cette faille réside dans le processus de publication automatique d'Open VSX, où un workflow GitHub Actions exécutait `npm install` sur du code d'extension non fiable, exposant un jeton secret (OVSX_PAT) avec des permissions pour publier ou écraser n'importe quelle extension. Les attaquants auraient pu voler ce jeton pour publier des mises à jour malveillantes, exposant des millions de développeurs à des attaques sur la chaîne d'approvisionnement, similaires à SolarWinds.
* Publication date : 2025/06/27
* 🌐 Source : hxxps[:]//securityaffairs[.]com/179398/hacking/taking-over-millions-of-developers-exploiting-an-open-vsx-registry-flaw.html
* 📦 CVE Impacted Product : Open VSX Registry (open-vsx[.]org)
* 📈 CVSS : Non spécifié, mais l'impact (prise de contrôle du marché, attaque de la chaîne d'approvisionnement, affectant des millions de développeurs) est **Critique**.
* 🛡️ Security recommandations : Les développeurs doivent être conscients des risques liés aux extensions d'IDE non vérifiées. Examiner les dépendances logicielles avec la même diligence que les packages d'autres sources. Renforcer la sécurité des processus CI/CD, notamment la gestion des secrets et l'exécution de code non fiable.
* 🌐 Indicator of Compromise :
    * DOMAIN : open-vsx[.]org
