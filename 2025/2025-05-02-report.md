## MintsLoader Drops GhostWeaver via Phishing, ClickFix — Uses DGA, TLS for Stealth Attacks
Un nouveau loader, MintsLoader, est utilisé dans des campagnes de phishing pour distribuer un cheval de Troie d'accès à distance (RAT) basé sur PowerShell nommé GhostWeaver. L'attaque utilise une chaîne d'infection en plusieurs étapes avec des scripts JavaScript et PowerShell obfusqués. Le malware intègre des techniques d'évasion de sandbox et de machine virtuelle, et utilise des DGA et TLS pour rester furtif. 🎣👻
*   Publication date : 2025/05/02
*   Source : https://thehackernews.com/2025/05/mintsloader-drops-ghostweaver-via.html
*   👾 Threat Tools : MintsLoader, GhostWeaver (RAT PowerShell), Scripts JS obfusqués, Scripts PowerShell obfusqués
*   📉 Threat Tactic : Phishing, Attaque en plusieurs étapes, Obfuscation, Évasion de sandbox/VM, DGA, TLS
*   🛡️ Security recommandations : Maintenir les systèmes et logiciels à jour, former les utilisateurs à la détection des e-mails de phishing, utiliser des solutions de sécurité avancées pour la détection des malwares et des RATs.

## Microsoft Sets Passkeys Default for New Accounts
Microsoft renforce la sécurité des nouveaux comptes en activant les passkeys par défaut, offrant ainsi une méthode d'authentification résistante au phishing. Cette initiative, annoncée un an après l'introduction du support des passkeys pour les comptes grand public, vise à réduire la dépendance aux mots de passe et à améliorer la protection contre les attaques par credential stuffing et force brute. 🔐✨
*   Publication date : 2025/05/02, 2025/05/02
*   Source : https://thehackernews.com/2025/05/microsoft-sets-passkeys-default-for-new.html, https://www.bleepingcomputer.com/news/microsoft/microsoft-makes-all-new-accounts-passwordless-by-default/
*   🛡️ Security recommandations : Activer l'authentification sans mot de passe (passkeys) lorsque possible, utiliser des méthodes d'authentification multi-facteurs fortes.

## Fake Security Plugin on WordPress Enables Remote Admin Access for Attackers
Une nouvelle campagne cible les sites WordPress en se faisant passer pour un plugin de sécurité légitime, "WP-antymalwary-bot.php". Ce malware offre aux attaquants un accès administrateur à distance, des fonctionnalités pour se cacher du tableau de bord admin, et la capacité d'exécuter du code à distance. Il inclut une fonction de "ping" pour communiquer avec un serveur de commande et de contrôle (C&C). 🚩🔌
*   Publication date : 2025/05/01
*   Source : https://thehackernews.com/2025/05/fake-security-plugin-on-wordpress.html
*   👾 Threat Tools : Plugin WordPress malveillant ("WP-antymalwary-bot.php")
*   📉 Threat Tactic : Compromission de site WordPress, Fausse extension/plugin, Accès à distance, Persistance, Exécution de code à distance, Communication C&C
*   🎯 Threat Target : Sites WordPress
*   🛡️ Security recommandations : Télécharger des plugins uniquement à partir de sources fiables, vérifier l'authenticité des plugins, maintenir WordPress et les plugins/thèmes à jour, utiliser des solutions de sécurité pour sites web.

## CVE-2025-3746 - WordPress One Tap Signin Plugin Authentication Bypass
Le plugin "OTP-less one tap Sign in" pour WordPress est vulnérable à une élévation de privilèges via une prise de contrôle de compte (account takeover) dans les versions 2.0.14 à 2.0.59. Le plugin ne valide pas correctement l'identité de l'utilisateur avant de mettre à jour ses détails, comme l'e-mail. Des attaquants non authentifiés peuvent ainsi modifier l'adresse e-mail d'utilisateurs arbitraires, y compris les administrateurs, et l'utiliser pour réinitialiser le mot de passe et accéder au compte. Le plugin renvoie également des cookies d'authentification exploitables. 🔑🔒
*   Publication date : 2025/05/02
*   Source : https://cvefeed.io/vuln/detail/CVE-2025-3746
*   🛡️ CVE : [CVE-2025-3746](https://nvd.nist.gov/vuln/detail/CVE-2025-3746)
*   📦 CVE IMPACTED PRODUCT : WordPress One Tap Signin Plugin (v2.0.14 to 2.0.59)
*   💯 CVSS : 9.8
*   📉 Threat Tactic : Élévation de privilèges, Prise de contrôle de compte, Contournement d'authentification, Modification d'adresse e-mail, Réinitialisation de mot de passe
*   🎯 Threat Target : Sites WordPress utilisant le plugin vulnérable
*   🛡️ Security recommandations : Désactiver ou supprimer le plugin affecté, attendre et appliquer un correctif sécurisé du vendeur, utiliser des méthodes d'authentification robustes.
*   Indicator of compromised :
    *   DOMAIN : plugins[.]trac[.]wordpress[.]org, www[.]wordfence[.]com, wordfence[.]com, cvefeed[.]io, ncvefeed[.]io
