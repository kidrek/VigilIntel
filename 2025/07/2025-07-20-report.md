### PoisonSeed Phishing : Bypasse l'authentification multifacteur FIDO2
Une campagne de phishing nommée PoisonSeed 🎣 parvient à contourner les protections des clés de sécurité FIDO2 en abusant de la fonction de connexion inter-appareils de WebAuthn. Les acteurs de la menace incitent les utilisateurs à approuver des demandes d'authentification de connexion depuis de faux portails d'entreprise, dégradant ainsi l'efficacité de la MFA basée sur FIDO2. Une autre technique observée implique l'enregistrement d'une clé FIDO par l'attaquant après un compromis de compte via phishing et une réinitialisation de mot de passe.
*   Publication date : 2025/07/19
*   Source : 🔗 https://www.bleepingcomputer.com/news/security/threat-actors-downgrade-fido2-mfa-auth-in-poisonseed-phishing-attack/
*   Threat Actor : 🕵️‍♂️ Campagne de phishing PoisonSeed
*   Threat Tactic : ⚔️ Phishing, Attaque de l'homme du milieu (AiTM), Ingénierie sociale pour dégrader l'authentification FIDO2, Vol de crédentiels
*   Threat Target : 🎯 Utilisateurs de l'authentification FIDO2 via WebAuthn, en particulier les portails d'entreprise utilisant des clés de sécurité.
*   Threat Tools : 🛠️ Backend AiTM, Faux portails de connexion, Code QR (pour l'authentification inter-appareils)
*   Security recommandations : 🔒 Sensibiliser les utilisateurs aux techniques de phishing avancées qui tentent de contourner les protections MFA résistantes au phishing. Appliquer des politiques de sécurité robustes pour la gestion des clés et la réinitialisation des mots de passe.

### Dégâts dans la chaîne d'approvisionnement : Des paquets npm populaires détournés par phishing
Les bibliothèques JavaScript populaires eslint-config-prettier et eslint-plugin-prettier ont été détournées cette semaine et transformées en droppers de logiciels malveillants 📦. Cette attaque de la chaîne d'approvisionnement a été réalisée via une attaque de phishing ciblée et le vol de références. Le mainteneur des paquets est tombé victime de phishing, permettant à une partie non autorisée d'accéder à son jeton npm et de publier des versions compromises.
*   Publication date : 2025/07/19
*   Source : 🔗 https://www.bleepingcomputer.com/news/security/popular-npm-linter-packages-hijacked-via-phishing-to-drop-malware/
*   Threat Actor : 🕵️‍♂️ Non spécifié, mais impliqué dans le phishing ciblé et le vol de crédentiels.
*   Threat Tactic : ⚔️ Attaque de la chaîne d'approvisionnement logicielle, Phishing ciblé, Vol de jetons npm, Déploiement de logiciels malveillants.
*   Threat Target : 🎯 Mainteneurs de paquets npm (JounQin), Développeurs et utilisateurs de `eslint-config-prettier`, `eslint-plugin-prettier`, `synckit`, `@pkgr/core`, `napi-postinstall`.
*   Threat Tools : 🛠️ `install.js` (script malveillant), `node-gyp.dll` (DLL malveillante), `rundll32` (pour exécution).
*   Security recommandations : 🔒 Marquer les versions affectées comme "dépréciées" sur les registres de paquets. Vérifier la légitimité de toutes les versions de paquets, surtout après des mises à jour inattendues. Renforcer la sécurité des comptes des mainteneurs (MFA).

### Vulnérabilité Fortinet FortiWeb CVE-2025-25257 Exploitée Activement
Des attaquants ont exploité une vulnérabilité critique de Fortinet FortiWeb, CVE-2025-25257, le jour même de la publication d'une preuve de concept (PoC) 🚨. Cette faille, avec un score CVSS de 9.6, est une injection SQL non authentifiée qui permet l'exécution de code à distance. Des dizaines de systèmes ont déjà été compromis. Les attaquants ont pu écrire des fichiers arbitraires en tant que `root`, menant à l'exécution de code Python via des fichiers `.pth` dans les répertoires CGI.

*   CVE Impacted Product : 🖥️ Fortinet FortiWeb (Versions antérieures à 7.6.4)
*   CVSS : 📈 9.6 (Critique)
*   Threat Tactic : ⚔️ Injection SQL, Exécution de code à distance (RCE), Exploitation de vulnérabilités un jour zéro.
*   Security recommandations : 🔒 Mettre à jour immédiatement Fortinet FortiWeb à la version 7.6.4 ou ultérieure. Surveiller activement les journaux et les systèmes pour détecter toute activité suspecte ou présence de webshells.

### Vulnérabilité d'Exécution de Code à Distance Microsoft SharePoint Server CVE-2025-53770
Une vulnérabilité critique d'exécution de code à distance 💥, CVE-2025-53770, a été découverte dans Microsoft SharePoint Server sur site. Cette faille résulte d'une désérialisation de données non fiables, permettant à un attaquant non autorisé d'exécuter du code sur le réseau. Microsoft est conscient qu'un exploit pour cette vulnérabilité existe dans la nature sous le nom de "ToolShell" et travaille sur une mise à jour corrective.
*   Publication date : 2025/07/20
*   Source : 🔗 https://cvefeed.io/vuln/detail/CVE-2025-53770
*   CVE : 🔗 CVE-2025-53770
*   CVE Impacted Product : 🖥️ Microsoft SharePoint Server (sur site)
*   CVSS : 📈 9.8 (Critique)
*   Threat Tactic : ⚔️ Désérialisation de données non fiables, Exécution de code à distance (RCE), Chaîne de vulnérabilités "ToolShell".
*   Security recommandations : 🔒 Appliquer la mitigation temporaire fournie par Microsoft dans la documentation CVE en attendant la publication du correctif complet. Surveiller les activités suspectes sur les serveurs SharePoint.

### Routeurs Tenda FH451 : Multiples Vulnérabilités de Dépassement de Tampon
Trois vulnérabilités critiques de dépassement de tampon basé sur la pile ont été découvertes dans le routeur Tenda FH451 1.0.0.9. Ces failles, identifiées comme CVE-2025-7853, CVE-2025-7854 et CVE-2025-7855, sont toutes notées avec un CVSS de 8.8 (Élevé). Elles affectent respectivement les fonctions `fromSetIpBind` dans `/goform/SetIpBind`, `fromVirtualSer` dans `/goform/VirtualSer`, et `fromqossetting` dans `/goform/qossetting`. La manipulation de l'argument 'page' dans chaque cas peut permettre une attaque à distance. Les exploits ont été divulgués publiquement.
*   Publication date : 2025/07/19 (pour les trois CVEs)
*   Source : 🔗 https://cvefeed.io/vuln/detail/CVE-2025-7853, 🔗 https://cvefeed.io/vuln/detail/CVE-2025-7854, 🔗 https://cvefeed.io/vuln/detail/CVE-2025-7855
*   CVE : 🔗 CVE-2025-7853, 🔗 CVE-2025-7854, 🔗 CVE-2025-7855
*   CVE Impacted Product : 🖥️ Tenda FH451 (version 1.0.0.9)
*   CVSS : 📈 8.8 (Élevé)
*   Threat Tactic : ⚔️ Dépassement de tampon basé sur la pile.
*   Security recommandations : 🔒 Mettre à jour le firmware du routeur Tenda FH451 dès qu'un correctif est disponible. Si un correctif n'est pas disponible, limiter l'exposition du routeur à Internet et mettre en place des mesures de segmentation réseau.

### WordPress WPLMS : Vulnérabilité d'Escalade de Privilèges
Le thème WPLMS pour WordPress est vulnérable à une escalade de privilèges (CVE-2015-10139) dans les versions 1.5.2 à 1.8.4.1. Via l'action AJAX `wp_ajax_import_data`, des attaquants authentifiés peuvent modifier des paramètres restreints et potentiellement créer un nouveau compte administrateur accessible.
*   Publication date : 2025/07/19
*   Source : 🔗 https://cvefeed.io/vuln/detail/CVE-2015-10139
*   CVE : 🔗 CVE-2015-10139
*   CVE Impacted Product : 🖥️ Thème WPLMS pour WordPress (versions 1.5.2 à 1.8.4.1)
*   CVSS : 📈 8.8 (Élevé)
*   Threat Tactic : ⚔️ Escalade de privilèges via action AJAX.
*   Security recommandations : 🔒 Mettre à jour le thème WPLMS vers une version corrigée. S'assurer que les comptes utilisateurs ont le niveau de privilège le plus bas nécessaire pour leurs fonctions.

### WordPress Work The Flow File Upload : Vulnérabilité Critique de Téléchargement Arbitraire de Fichiers
Le plugin Work The Flow File Upload pour WordPress est vulnérable aux téléchargements de fichiers arbitraires (CVE-2015-10138) dans les versions jusqu'à 2.5.2 incluses. Cette faille est due à un manque de validation du type de fichier dans le serveur et les fichiers de test de `jQuery-File-Upload-9.5.0`. Cela permet à des attaquants non authentifiés de télécharger des fichiers arbitraires sur le serveur, ce qui peut rendre possible l'exécution de code à distance.
*   Publication date : 2025/07/19
*   Source : 🔗 https://cvefeed.io/vuln/detail/CVE-2015-10138
*   CVE : 🔗 CVE-2015-10138
*   CVE Impacted Product : 🖥️ Plugin Work The Flow File Upload pour WordPress (versions jusqu'à 2.5.2 incluses)
*   CVSS : 📈 9.8 (Critique)
*   Threat Tactic : ⚔️ Téléchargement arbitraire de fichiers, Exécution de code à distance (RCE).
*   Security recommandations : 🔒 Mettre à jour le plugin Work The Flow File Upload vers une version corrigée. Si une mise à jour n'est pas disponible, désactiver ou supprimer le plugin. Mettre en place des validations de types de fichiers strictes au niveau du serveur.

### VMware corrige les vulnérabilités Zero-Day exploitées à Pwn2Own Berlin
VMware a corrigé quatre vulnérabilités zero-day ☁️ affectant plusieurs de ses produits de virtualisation, notamment ESXi, Workstation, Fusion et VMware Tools. Ces failles ont été exploitées lors du concours de sécurité Pwn2Own Berlin 2025, où des chercheurs ont démontré des exécutions de code arbitraire et des évasions de machines virtuelles. Bien qu'aucune exploitation active dans la nature n'ait été détectée, ces vulnérabilités posent des risques critiques pour les environnements d'entreprise.
*   Publication date : 2025/07/19
*   Source : 🔗 https://thecyberthrone.in/2025/07/19/vmware-fixes-vulnerabilities-exploited-at-pwn2own-berlin/
*   CVE : 🔗 CVE-2025-41236, 🔗 CVE-2025-41237, 🔗 CVE-2025-41238, 🔗 CVE-2025-41239
*   CVE Impacted Product : 🖥️ VMware ESXi, Workstation, Fusion, VMware Tools
*   CVSS : 📈 Non spécifié, mais implicitement critique en raison de l'exploitation Pwn2Own (RCE/évasion VM).
*   Threat Tactic : ⚔️ Exécution de code à distance (RCE), Évasion de machine virtuelle, Exploitation de vulnérabilités zero-day.
*   Security recommandations : 🔒 Appliquer sans délai les mises à jour de sécurité pertinentes pour ESXi, Workstation, Fusion et VMware Tools via le portail Customer Connect de VMware ou les workflows de gestion de la configuration. Prioriser la mitigation des risques, surtout dans les environnements hébergeant des charges de travail sensibles.

### Nouvelle vulnérabilité Zero-Day CrushFTP exploitée dans la nature
Une faille zero-day critique 🔓 a été confirmée dans la plateforme de transfert de fichiers gérés CrushFTP. L'exploitation active a commencé le 18 juillet 2025. La vulnérabilité est liée à une logique de validation incomplète. En l'absence de déploiement de la fonction de proxy DMZ, l'exploit accorde des privilèges d'administrateur, permettant aux attaquants de créer de nouveaux utilisateurs, d'exfiltrer des données ou de se déplacer latéralement. Les honeypots Shadowserver ont déjà enregistré des tentatives d'exploitation.
*   Publication date : 2025/07/19
*   Source : 🔗 https://cybersecuritynews.com/crushftp-0-day-vulnerability/
*   CVE : 🔗 Pas de CVE attribué au moment de la publication (Zero-Day)
*   CVE Impacted Product : 🖥️ CrushFTP (Versions antérieures à 11.3.4_26)
*   CVSS : 📈 Non spécifié, mais implicitement critique (Zero-day, accès administrateur, exploitation active).
*   Threat Actor : 🕵️‍♂️ Groupes de ransomwares et acteurs d'espionnage (général).
*   Threat Tactic : ⚔️ Contournement de la logique de validation, Élévation de privilèges, Exécution de code à distance, Accès non autorisé.
*   Security recommandations : 🔒 Mettre à jour immédiatement CrushFTP à la build 11.3.4_26. Mettre en place une gestion rigoureuse des correctifs, une segmentation réseau et une revue vigilante des journaux.
