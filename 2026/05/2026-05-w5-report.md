## Strategic Recommendations

1. **Enforce FIDO2 Hardware MFA Globally**
   Traditional SMS and push-based MFA are routinely bypassed by sophisticated Adversary-in-the-Middle (AiTM) phishing techniques. Deploy FIDO2/WebAuthn hardware tokens (e.g., YubiKey 5 Series) globally for all identity providers, especially administrative and developer environments, to mitigate persistent credential harvesting campaigns run by actors like Lazarus.

2. **Establish Zero-Trust Edge Segmentation**
   Adversaries like Volt Typhoon target public-facing appliances to establish initial network footholds. Segment all edge network devices (VPNs, firewalls, and SD-WAN hubs) from internal enterprise environments using strict access control lists (ACLs) and stateful inspection. Implement daily configuration compliance verification to prevent unauthorized modifications.

## Operational Recommendations

1. **Patch Ivanti Secure Gateways Immediately**
   Run the official Ivanti Integrity Checker Tool (ICT) on all active instances to check for indicators of compromise (IoCs) associated with CVE-2026-31835. Apply version 22.6R2.1 or higher immediately if no anomalies are detected. Treat any anomalous outbound LDAP or HTTPS connections from the gateway as an active intrusion.

2. **Restrict and Monitor Inbound RDP**
   Analyze Active Directory Event ID 4624 (successful logon) and Event ID 4625 (failed logon) looking for anomalous remote desktop connection attempts over port 3389. Restrict RDP access strictly to local subnets or require secure VPN tunnels with multi-factor authentication to block lateral movement vectors used by Black Basta.

## Recommandations Stratégiques

1. **Imposer l'authentification FIDO2 par clé matérielle**
   Les MFA traditionnels par SMS ou notifications push sont fréquemment contournés par des attaques d'ingénierie sociale de type Adversary-in-the-Middle (AiTM). Déployez des clés matérielles FIDO2/WebAuthn (ex. YubiKey 5) pour l'ensemble des accès à l'infrastructure d'entreprise pour contrer les campagnes de vol de session menées par Lazarus.

2. **Établir une segmentation périphérique Zero-Trust**
   Des adversaires tels que Volt Typhoon ciblent les équipements périmétriques pour s'implanter. Isolez les passerelles VPN, pare-feux et routeurs SD-WAN du cœur de réseau à l'aide de listes de contrôle d'accès (ACL) restrictives et d'une inspection à états afin d'empêcher tout mouvement latéral en cas de compromission.

## Recommandations Opérationnelles

1. **Corriger immédiatement les passerelles Ivanti**
   Exécutez l'outil officiel d'intégrité d'Ivanti (ICT) sur toutes les instances actives pour rechercher des indicateurs de compromission liés à la CVE-2026-31835. Appliquez la mise à jour 22.6R2.1 ou supérieure. Considérez toute connexion LDAP ou HTTPS sortante anormale de la passerelle comme une intrusion active.

2. **Restreindre et surveiller le trafic RDP**
   Analysez les événements Active Directory (ID 4624 et 4625) à la recherche de connexions RDP anormales via le port 3389. Bloquez le protocole RDP provenant d'Internet et limitez-le strictement à des segments locaux sécurisés par VPN avec double facteur, neutralisant ainsi les tactiques de déplacement de Black Basta.
