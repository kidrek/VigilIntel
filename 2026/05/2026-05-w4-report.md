## Strategic Recommendations

1. **Enforce FIDO2 Hardware MFA for Developer Accounts**
   Traditional SMS or software TOTP MFA is actively bypassed by advanced groups like Screening Serpens and OtterCookie utilizing automated session-hijacking frameworks. Deploy FIDO2/WebAuthn physical hardware keys (YubiKey 5 Series, Google Titan) for all administrative profiles, cloud tenants, and source control management systems. This completely mitigates the risk of credential theft via malicious VSCode extensions or active OIDC token harvesting.

2. **Implement Automated Software Composition Analysis Tools**
   Supply chain compromises in public registries (npm, PyPI) used by developers, such as the 'art-template' and 'Shai-Hulud' campaigns, easily bypass static code analysis. Integrate automated Software Composition Analysis (SCA) solutions like Snyk or GitHub Advanced Security directly into CI/CD pipelines. This ensures that vulnerable packages and malicious preinstall scripts are programmatically blocked before compilation.

## Operational Recommendations

1. **Audit IIS Web Modules for BadIIS Infection**
   The BadIIS backdoor module manipulates search engine crawler traffic to perform SEO fraud and harvest sensitive information. Run the IIS management command 'appcmd list modules' on all Windows IIS hosts to identify active modules, cross-referencing them against known baselines. Immediately remove any unapproved DLL files from 'C:\Windows\System32\inetsrv\' and restrict write access to 'Web.config' for the 'IIS_IUSRS' service account.

2. **Disable Unused Former Employee Access Credentials**
   Adversaries exploit active access credentials belonging to former employees to gain initial network entry without triggering security alarms. Run a PowerShell script against Active Directory and Okta to correlate the 'terminationDate' field from HR databases against active user sessions. Immediately disable all mismatched accounts, revoke active OAuth tokens, and delete registered SSH keys.

- **Santé :** NYC Health + Hospitals a subi une fuite de 1,8 million de dossiers patients contenant des données biométriques, en parallèle d'une seconde violation chez Radiology Associates of Richmond touchant 266 000 patients.
- **Technologie et Télécoms :** Trump Mobile a exposé les coordonnées physiques et numéros de téléphones de ses clients via un bucket cloud mal configuré, tandis que STC en Arabie Saoudite a vu ses plages IP compromises pour héberger 72 % des serveurs C2 du Moyen-Orient.
- **Finance :** La plateforme de finance décentralisée Verus DeFi a subi un vol de crypto-actifs d'une valeur de 8,5 millions de dollars, restitués par la suite à la suite d'un accord amiable.
- **Infrastructures critiques :** Des automates industriels Rockwell Automation et Allen-Bradley gérant la distribution d'eau et d'énergie ont été ciblés de manière intensive par le groupe étatique CyberAv3ngers.
- **Éducation :** La plateforme d'apprentissage Canvas LMS au Canada a été compromise, exposant les adresses e-mail et profils de cours de 1 700 étudiants et enseignants.

## Tendances & Signaux Faibles

**Techniques émergentes :**
- Détournement d'AppDomainManager pour désactiver la journalisation .NET ETW
- Compromission de chaîne d'approvisionnement via des extensions d'IDE (VSCode)
- Attaques de type watering-hole exploitant des kits iOS (Coruna) via des packages npm piégés
- Ransomwares sans binaire local ciblant les partages réseau SMB exposés (WantToCry)

**Techniques en déclin :**
- Credential stuffing classique sans optimisation par API de recherche
- Pièces jointes malveillantes simples sans validation sandbox multi-étapes

**Contexte géopolitique :** Les tensions géopolitiques continuent d'orienter les capacités cyber-offensives vers les entreprises privées. Au Moyen-Orient, les groupes iraniens Screening Serpens et MuddyWater exploitent des réseaux d'opérateurs nationaux et des outils d'IA pour cibler l'aviation occidentale. Parallèlement, le conflit en Ukraine alimente un espionnage constant par UAC-0057 et des représailles DDoS par NoName057(16) contre les infrastructures européennes.

## Recommandations Stratégiques

1. **Imposer la double authentification matérielle FIDO2**
   Les authentifications double facteur par SMS ou applications logicielles sont contournées par des groupes avancés comme Screening Serpens et OtterCookie. Déployez des clés matérielles physiques FIDO2/WebAuthn (YubiKey 5, Google Titan) pour tous les comptes d'administration, d'infrastructure cloud et de gestion de dépôts. Cela neutralise le vol de session lié aux extensions VSCode ou à l'exfiltration de jetons OIDC.

2. **Déployer des outils d'analyse de composition logicielle**
   Les attaques de chaîne d'approvisionnement ciblant les registres publics (npm), à l'instar des campagnes 'art-template' et 'Shai-Hulud', contournent les analyses statiques classiques. Intégrez des outils d'analyse de composition logicielle (SCA) comme Snyk ou GitHub Advanced Security au sein de vos pipelines de build CI/CD pour bloquer les paquets compromis avant compilation.

## Recommandations Opérationnelles

1. **Auditer les modules IIS contre BadIIS**
   Le module malveillant BadIIS manipule le trafic des robots de recherche pour commettre de la fraude SEO et exfiltrer des données. Exécutez la commande d'administration IIS 'appcmd list modules' sur vos serveurs Web IIS pour identifier les modules actifs et les comparer à votre référentiel. Supprimez toute DLL suspecte dans 'C:\Windows\System32\inetsrv\' et verrouillez l'accès au fichier 'Web.config'.

2. **Désactiver les comptes d'anciens employés inactifs**
   Les attaquants exploitent fréquemment les identifiants actifs d'anciens collaborateurs pour pénétrer les réseaux sans déclencher d'alertes. Exécutez un script PowerShell corrélant votre Active Directory/Okta avec les bases de données RH pour identifier les comptes d'employés partis. Désactivez immédiatement ces comptes, révoquez leurs jetons OAuth et supprimez leurs clés SSH.
