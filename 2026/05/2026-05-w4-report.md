## Strategic Recommendations

1. **Enforce FIDO2 Hardware MFA**
   Traditional SMS and TOTP MFA controls are bypassed using advanced phishing kits and LDAP attribute exploits observed this period (like CVE-2024-12802 in SonicWall Gen6). Deploy FIDO2/WebAuthn hardware keys (such as YubiKey 5 or Google Titan) for all cloud provider consoles, Active Directory, and critical VPN gateways. This mitigates credential-stuffing and session hijacking risks across the enterprise.

2. **Deploy Private NPM Registries**
   Software supply chain attacks (e.g., TeamPCP's Shai-Hulud npm worm and malicious VSCode extensions) easily bypass SLSA Build Level 3 signatures. Transition all development teams to private registries (JFrog Artifactory or Sonatype Nexus) with mandatory Software Composition Analysis (SCA) blocking malicious packages. This restricts unauthorized package updates and outbound OIDC token leakage.

## Operational Recommendations

1. **Enable Defender Tamper Protection**
   Threat groups such as 'The Gentlemen' ransomware utilize PowerShell scripts (Set-MpPreference) to disable Defender real-time protection during intrusions. Enforce Tamper Protection globally via Microsoft Intune or Group Policy Objects (GPO) to prevent local administrative bypasses. Monitor Event ID 5001 to detect unauthorized attempts to modify antivirus states.

2. **Block Port 445 SMB Globally**
   Ransomware campaigns like WantToCry opportunistically scan for exposed port 445 SMB interfaces to execute fileless encryption over the network. Implement immediate outbound and inbound edge firewall rules to drop all external TCP/UDP port 445 traffic. Transition necessary remote connections to IPsec VPN tunnels with rigorous access control lists (ACLs).

## Recommandations Stratégiques

1. **Imposer le MFA matériel FIDO2**
   Les mécanismes d'authentification MFA classiques par SMS ou TOTP sont contournés par des kits de phishing avancés et des failles LDAP observées cette période (comme CVE-2024-12802 de SonicWall). Déployez des clés matérielles FIDO2/WebAuthn (YubiKey 5, Google Titan) pour tous les accès cloud et VPN critiques afin de neutraliser le vol de sessions.

2. **Déployer des registres npm privés**
   Les attaques de chaîne d'approvisionnement (comme le ver npm Shai-Hulud de TeamPCP) contournent les signatures de niveau SLSA Build Level 3. Imposez l'utilisation de registres privés (JFrog Artifactory, Sonatype Nexus) couplés à des outils d'analyse SCA pour inspecter systématiquement les dépendances et bloquer l'exfiltration de jetons OIDC.

## Recommandations Opérationnelles

1. **Activer l'anti-altération Defender**
   Des groupes comme 'The Gentlemen' utilisent PowerShell (Set-MpPreference) pour couper la protection Defender. Imposez la 'Tamper Protection' de manière centralisée via Intune ou GPO pour empêcher toute désactivation locale par script, et surveillez l'Event ID 5001 dans votre SIEM.

2. **Bloquer le port SMB 445**
   Des ransomwares comme WantToCry scannent activement les partages réseau port 445 exposés pour chiffrer les fichiers à distance sans binaire local. Configurez immédiatement des règles strictes sur vos pare-feux externes pour bloquer tout flux entrant/sortant sur le port 445 et redirigez les flux nécessaires vers des tunnels VPN IPsec chiffrés.
