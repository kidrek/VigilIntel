## Trends & Weak Signals

**Emerging techniques:**
- Pure Extortion without data encryption to minimize active defense triggers (ShinyHunters)
- AppDomainManager Hijacking to blind .NET ETW logging and bypass endpoint detection (Screening Serpens)
- Watering-Hole attacks via npm dependency tag hijacking (art-template & Laravel Lang)

**Declining techniques:**
- Traditional noisy ransomware with localized bulk endpoint encryption is being replaced by stealthy data-exfiltration-only campaigns.
- Simple SMS and email-based TOTP MFA are declining in security efficiency due to the rise of active session-hijacking tools.

**Geopolitical context:** Tensions in Eastern Europe and the Middle East are directly reflected in cyberspace. Simultaneously, the European Union's focus turns toward securing physical and logical submarine networks and Arctic communication cables.

## Strategic Recommendations

1. **Enforce FIDO2 Hardware MFA on Cloud**
   Legacy SMS/TOTP MFA is routinely bypassed by advanced phishing and identity attacks (such as the ROADtools tactics and Screening Serpens campaigns observed this week). Implement FIDO2/WebAuthn hardware tokens (e.g., YubiKey 5) to secure all Entra ID administrative accounts. This mitigates token theft and unauthorized device registration.

2. **Secure Software Supply Chain Dependency Versions**
   Cybercriminals like TeamPCP and the developers of the Coruna exploit kit are actively poisoning npm and Composer registries (affecting art-template and Laravel Lang). Enforce Composer and npm lockfile verification (composer.lock, package-lock.json) with strict SHA-384/512 integrity hashes in your CI/CD pipelines. This blocks malicious code injections at the build level.

## Operational Recommendations

1. **Patch Drupal Core SQL Injection Vulnerability**
   Threat actors are actively exploiting CVE-2026-9082 to compromise PostgreSQL-backed Drupal deployments. Execute database audits and update all Drupal Core instances to versions 11.3.10+ or 10.6.9+ immediately. Configure web application firewall (WAF) rules to filter SQL syntax in HTTP requests to block pending attacks.

2. **Audit IIS Web Servers for BadIIS**
   Cisco Talos revealed active campaigns using the BadIIS module for SEO poisoning and backdoor access. Run the command 'appcmd list modules' on all corporate Microsoft IIS servers to inventory authorized extensions. Remove any unrecognized DLL files inside the system32\inetsrv directory and restrict IIS_IUSRS write privileges.

Cela bloque l'injection de modules malveillants.

## Recommandations Opérationnelles

1. **Appliquer le correctif Drupal Core SQLi**
   Des attaquants exploitent activement la faille CVE-2026-9082 pour compromettre les déploiements Drupal s'appuyant sur PostgreSQL. Auditez vos bases de données et mettez à jour Drupal Core vers les versions 11.3.10+ ou 10.6.9+ d'urgence. Configurez des règles WAF pour filtrer la syntaxe SQL et bloquer les tentatives d'intrusion.

2. **Auditer les serveurs web Microsoft IIS**
   Cisco Talos a révélé des campagnes actives utilisant le malware BadIIS pour de la fraude SEO et du contrôle persistant. Exécutez la commande 'appcmd list modules' sur l'ensemble de vos serveurs IIS pour lister les extensions autorisées. Supprimez toute DLL suspecte dans le répertoire system32\inetsrv et restreignez les privilèges d'écriture d'IIS_IUSRS.
