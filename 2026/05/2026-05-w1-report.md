## Strategic Recommendations

1. **Transition to FIDO2 Passwordless Authentication**
   As evidenced by the EvilTokens and ConsentFix v3 campaigns, traditional MFA is frequently bypassed via OAuth session theft. Deploy FIDO2/WebAuthn hardware keys (e.g., YubiKey 5) to eliminate reliance on vulnerable session tokens and device codes. This mitigates the risk of account takeover even if users interact with phishing sites.

2. **Implement Software Supply Chain Guardrails**
   The Shai-Hulud worm deployed by TeamPCP proves that public registries (npm) are a high-risk vector. Use private package managers (Artifactory, Nexus) with strict 'lock-down' periods for new packages (72h quarantine) and mandate the use of '--ignore-scripts' during local development installations to block preinstall hooks.

## Operational Recommendations

1. **Patch cPanel and WHM (CVE-2026-41940)**
   Immediate update to cPanel version 11.136.0.5 or higher is required to close the critical authentication bypass exploited by Sorry Ransomware. For unpatched systems, restrict access to ports 2083 and 2087 to trusted IP ranges only via host-based firewalls.

2. **Audit Entra ID Application Consents**
   Run the Microsoft Graph PowerShell command 'Get-MgServicePrincipal' to identify apps with 'User.Read.All' or 'Mail.Read' permissions granted recently. Revoke any unrecognized service principals and use 'Revoke-MgUserSignInSession' for affected users to invalidate exfiltrated OAuth tokens.
