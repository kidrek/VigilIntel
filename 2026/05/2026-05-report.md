## Executive Summary

## Strategic Recommendations

1. **Enforce FIDO2 Hardware MFA for All High-Value Identities**
   Traditional MFA (SMS/TOTP) is systematically bypassed by AiTM kits like Bluekit and EvilTokens observed this month. Deploy FIDO2 hardware keys (e.g., YubiKey 5) for all developers and administrators to prevent session hijacking and OIDC token theft used by TeamPCP and ShinyHunters.

2. **Adopt Zero-Trust Supply Chain Architecture (SBOM/SCA)**
   Given the 'wormification' of npm/PyPI by TeamPCP, implement mandatory Software Bill of Materials (SBOM) and automated SCA to pin dependencies to SHA-256 hashes. Scan CI/CD pipelines for unauthorized 'preinstall' hooks and Bun runtime payloads to mitigate self-propagating worms.

## Operational Recommendations

1. **Immediate Patching of Edge Security and Linux Kernels**
   Prioritize patching PAN-OS (CVE-2026-0300), cPanel (CVE-2026-41940), and Ivanti (CVE-2026-6973). These are in the CISA KEV and are under active exploitation by state actors for unauthenticated root access.

2. **Detect eBPF Persistence and Audit Git Hooks**
   Use 'bpftool prog show' to identify unauthorized eBPF programs associated with the QLNX rootkit. Additionally, audit local '.git/hooks' directories for scripts containing curl/wget commands used by Lazarus Group to deploy InvisiFerret.
