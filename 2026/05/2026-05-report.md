## Threat Level

## Executive Summary

## Trends & Weak Signals

**Emerging techniques:**
- Poisoning official VS Code IDE extensions (such as Nx Console) to execute silent background exfiltrations
- Abusing the OAuth 2.0 Device Authorization Grant Flow (Device Code) to bypass MFA
- Targeting developer settings files (~/.claude/settings.json) to harvest private AI API keys
- Real-time interception of one-time passwords (OTP) via synchronized PhaaS panels
- BYOVD (Bring Your Own Vulnerable Driver) deployments using signed drivers to kill EDR agents

**Declining techniques:**
- Traditional SMS or TOTP based multi-factor authentication due to rapid, automated session hijacking
- Static signature-based detection for newly compiled open-source binaries (Go/Rust)
- Volume-only ransomware encryption without pre-exfiltration of sensitive data

**Geopolitical context:** Geopolitical friction heavily steered operations, with Russian groups executing synchronized cyber-sabotage against European logistics and Polish utility networks, while North Korea accelerated supply chain intrusions to bypass financial sanctions. Concurrently, Middle Eastern regional conflicts manifested in cyber espionage campaigns targeting aerospace and telecommunications.

## Strategic Recommendations

1. **Enforce Phishing-Resistant FIDO2 Hardware MFA**
   Traditional SMS/TOTP MFA is routinely bypassed by advanced OAuth device-code phishing (abused by Kali365) and session hijacking (ACR Stealer). Deploy FIDO2/WebAuthn hardware tokens (such as YubiKey 5) to secure developer accounts and administrators accessing cloud resources. Restrict alternative authentication protocols to prevent fallback exploits.

2. **Establish Secure Mirroring for Dependency Registries**
   With actors like TeamPCP poisoning npm/PyPI registries and VS Code extensions using falsified Sigstore provenance, pipelines must enforce strict lockfile integrity. Integrate Static Application Security Testing (SAST) and Software Composition Analysis (SCA) to analyze dependency trees. Require all external libraries to be validated inside private enterprise proxies like Nexus.

## Operational Recommendations

1. **Disable OAuth Device Code Authorization Flow**
   To block Kali365 and related PhaaS platforms from hijacking Microsoft 365 sessions, immediately disable the Device Code flow in Microsoft Entra ID. Configure conditional access policies restricting OAuth authorization to corporate-managed and compliant devices. Revoke any active sessions where anomalous user-agent patterns (e.g. python-requests) are detected.

2. **Apply Hotfixes for Ghost CMS SQLi**
   To remediate active ClickFix exploitation (CVE-2026-26980), immediately update all Ghost CMS instances to version 6.19.1 or higher. Restrict local database user permissions to read-only for public directories and deploy Web Application Firewall (WAF) rules to detect SQL injections. Purge any unmapped .js scripts from the web directory.

## Secteurs & Géographies Ciblés

**Secteurs (classification) :** Government_and_Defense, Education_and_Research, Technology_and_Telecom, Finance, Critical_Infrastructure, Healthcare_and_Life_Sciences, Retail_and_Consumer, Hospitality_and_Entertainment, Industry_and_Manufacturing, Business_Services, Real_Estate_and_Construction

**Pays :** France, États-Unis, Italie, Brésil, Japon, Australie, Canada, Allemagne, Pologne, Ukraine

**Panorama détaillé :**

- **Secteur public & Gouv :** Violations majeures incluant l'ANTS (France) exposant 19M de dossiers, la fuite de clés CISA sur GitHub, et la compromission d'infrastructures italiennes par Salt Typhoon.
- **Éducation :** Instructure (Canvas LMS) a subi une exfiltration monumentale touchant 280M d'enregistrements d'étudiants dans 330 universités mondiales pendant la période des examens.
- **Santé :** NYC Health + Hospitals compromise, avec exfiltration des dossiers de 1.8M de patients ; Medtronic a notifié 9M de patients d'une exposition de données.
- **Finance :** La fintech Addi a subi une fuite de 34M de comptes, la plateforme de carding B1ack's Stash a exposé 4.6M de cartes, et le pont Verus DeFi a perdu 8.5M de dollars.
- **Technologie :** Les dépôts de code d'OpenAI, Mistral AI et Trellix ont été ciblés par TeamPCP et RansomHouse via le piratage d'extensions d'IDE.
- **Infrastructures critiques :** Sabotages de réseaux ferroviaires et d'eau, incluant des injections radio sur le TGV taïwanais et des compromissions de stations d'eau polonaises par APT28.
- **Distribution & Tourisme :** ShinyHunters a compromis 600k comptes de franchisés 7-Eleven, et une attaque supply chain contre le prestataire Itea a exposé les historiques de Belambra et Pierre & Vacances.
- **Industrie & Fabrication :** Foxconn a été touché par le rançongiciel Nitrogen, entraînant la fuite de 8 To de plans techniques confidentiels appartenant à des clients majeurs de l'électronique.

## Tendances & Signaux Faibles

**Techniques émergentes :**
- Empoisonnement d'extensions d'IDE VS Code (ex: Nx Console) pour réaliser des exfiltrations en arrière-plan
- Abus du flux d'autorisation OAuth 2.0 Device Code pour contourner le MFA
- Ciblage des configurations de développeurs (~/.claude/settings.json) pour dérober des clés API d'IA privées
- Interception en temps réel des mots de passe à usage unique (OTP) via des panels de PhaaS synchronisés
- Déploiement BYOVD (Bring Your Own Vulnerable Driver) via des pilotes légitimes pour arrêter les agents EDR

**Techniques en déclin :**
- Authentification multifacteur par SMS/TOTP traditionnelle face à l'essor du piratage de session
- Détection par signature statique pour les binaires open-source compilés dynamiquement (Go/Rust)
- Chiffrement par rançongiciel simple sans exfiltration préalable de données sensibles

**Contexte géopolitique :** Les tensions géopolitiques ont fortement influencé les opérations, les groupes russes menant des cyber-sabotages synchronisés contre la logistique européenne et les services polonais, tandis que la Corée du Nord a intensifié ses intrusions supply chain pour contourner les sanctions. Parallèlement, le conflit au Moyen-Orient s'est traduit par des campagnes d'espionnage ciblant l'aérospatiale et les télécoms.

## Recommandations Stratégiques

1. **Imposer l'authentification MFA matérielle FIDO2**
   Les méthodes MFA traditionnelles (SMS/TOTP) sont régulièrement contournées par des attaques de phishing OAuth Device Code (exploitées par Kali365) et le vol de session (ACR Stealer). Déployez des clés matérielles FIDO2/WebAuthn (type YubiKey 5) pour sécuriser les comptes de développeurs et d'administrateurs. Restreignez les protocoles d'authentification alternatifs pour empêcher tout contournement.

2. **Sécuriser les pipelines CI/CD par signature cryptographique**
   Face aux attaques de TeamPCP empoisonnant les registres npm/PyPI et les extensions VS Code avec des signatures Sigstore falsifiées, les pipelines d'intégration doivent imposer un contrôle sytématique des fichiers lock. Intégrez des outils SAST et SCA pour analyser les arbres de dépendances et validez l'intégrité des bibliothèques via des serveurs de mise en cache privés type Nexus.

## Recommandations Opérationnelles

1. **Désactiver le flux d'autorisation OAuth Device Code**
   Afin de bloquer Kali365 et les plateformes de PhaaS associées qui détournent les sessions Microsoft 365, désactivez immédiatement le flux Device Code dans Microsoft Entra ID. Configurez des règles d'accès conditionnel limitant l'authentification OAuth aux terminaux approuvés et gérés par l'entreprise. Révoquez les sessions actives présentant des user-agents suspects comme python-requests.

2. **Appliquer les correctifs pour Ghost CMS**
   Pour contrer les attaques actives de la campagne ClickFix (CVE-2026-26980), mettez à jour d'urgence l'ensemble des instances de Ghost CMS vers la version 6.19.1 ou supérieure. Limitez les privilèges de l'utilisateur de base de données et déployez des règles WAF pour bloquer les tentatives d'injections SQL. Supprimez tout script .js non identifié à la racine du serveur web.
