## Strategic Recommendations

1. **Enforce FIDO2 Hardware MFA for Developer Accounts**
   Traditional SMS/TOTP MFA is bypassed by EvilTokens Device Code phishing and token theft (used by ShinyHunters and TCLBanker this period). Deploy FIDO2/WebAuthn hardware keys (YubiKey 5, Google Titan) for all accounts with access to cloud infrastructure and CI/CD pipelines.

2. **Implement Kernel Integrity Monitoring for Linux Fleet**
   Threats like QLNX and 'Dirty Frag' (CVE-2026-43284) bypass traditional detection. Use tools like 'bpftool' to monitor loaded eBPF programs and enforce signed kernel modules to prevent rootkits and unauthorized privilege escalation.

## Operational Recommendations

1. **Audit and Baseline Linux PAM Modules Immediately**
   Actors like darkworm (PamDOORa) and QLNX are modifying /etc/pam.d/ and /lib/security/ to capture credentials. Use 'debsums' or 'rpm -V' to verify the integrity of authentication libraries and alert on any unauthorized modifications.

2. **Block ADB Port 5555 on Perimeter Firewalls**
   The xlabs_v1 Mirai variant is actively scanning for exposed Android Debug Bridge (ADB) ports to build a DDoS botnet. Restrict port TCP/5555 to internal management IPs only and disable ADB on all Android TV and IoT devices by default.

1. **Imposer la MFA matérielle FIDO2 pour les comptes développeurs**
