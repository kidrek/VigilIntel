## Strategic Recommendations

1. **Enforce FIDO2 Hardware MFA for Developer Accounts**
   Traditional SMS/TOTP MFA is bypassed by EvilTokens Device Code phishing used by TeamPCP and Scattered Spider. Deploy FIDO2 hardware keys (YubiKey 5) for all accounts with access to cloud infrastructure and repositories.

2. **Establish an SBOM Auditing Program**
   Supply chain poisoning (TanStack/node-ipc) is now a primary vector. Implement automated Software Bill of Materials (SBOM) scanning using CycloneDX and tools like Syft to detect malicious dependencies in CI/CD pipelines.

## Operational Recommendations

1. **Patch Cisco SD-WAN controllers (CVE-2026-20182) immediately**
   Given the CVSS 10 score and active exploitation by UAT-8616, update all Catalyst SD-WAN components to the latest version. Restrict NETCONF/SSH access to known management IPs only.

2. **Disable VS Code Automatic Task Execution**
   Attackers are abusing 'runOn: folderOpen' in tasks.json. Set 'task.allowAutomaticTasks' to 'off' in global VS Code settings and audit all workspace-level configurations before opening projects.

1. **Imposer le MFA matériel FIDO2 pour les développeurs**
   Le MFA traditionnel est contourné par le phishing de type Device Code utilisé par TeamPCP. Déployez des clés matérielles FIDO2 (YubiKey 5) pour tous les accès aux infrastructures cloud et dépôts.

2. **Établir un programme d'audit SBOM**

## Recommandations Opérationnelles

1. **Patcher les contrôleurs Cisco SD-WAN (CVE-2026-20182)**
   Vu le score CVSS 10 et l'exploitation active par UAT-8616, mettez à jour tous les composants Catalyst SD-WAN. Restreignez l'accès NETCONF/SSH aux IPs de gestion connues.

2. **Désactiver l'exécution automatique des tâches VS Code**
   Les attaquants abusent de 'runOn: folderOpen'. Réglez 'task.allowAutomaticTasks' sur 'off' dans les paramètres globaux de VS Code et auditez les configurations avant d'ouvrir un projet.
