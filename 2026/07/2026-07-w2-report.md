## Strategic Recommendations

1. **Deploy FIDO2 Hardware Keys for Code Repositories**
   Recent software supply-chain attacks, notably the compromise of the Jscrambler npm package and the CISA GitHub leakage, show that threat actors are aggressively targeting developer environments. Organizations must transition from traditional SMS or software-based MFA to FIDO2/WebAuthn hardware tokens (e.g., YubiKey 5 Series) to secure developer accounts and CI/CD pipelines. This direct control prevents credential harvesting by infostealers like LummaC2 and protects access to proprietary source code.

2. **Enforce Host-Only Network Binding for AI Middleware**
   With the emergence of scanning campaigns targeting Model Context Protocol (MCP) servers and Cursor/Claude credentials, exposing AI orchestration layers on public IP addresses presents a critical backdoor. Force all local MCP instances and AI agent endpoints to bind exclusively to the loopback interface (127.0.0.1) or restrict traffic through authenticated reverse proxies with mTLS enabled. This mitigation completely blocks unauthorized remote JSON-RPC execution and prompt injections like Ghostcommit.

## Operational Recommendations

1. **Audit and Terminate Vulnerable Progress ShareFile Controllers**
   Following the urgent directive from Progress Software regarding an active threat targeting Storage Zone Controllers (associated with CVE-2023-24489 and CVE-2026-50656), administrators must immediately power down affected Windows servers. Execute the PowerShell command `Stop-Service -Name 'CitrixShareFileStorageZones'` to safely halt the service and isolate the servers at the firewall level. Do not restart the instances until the official vendor patches are deployed and the underlying IIS directory has been audited for `/cifs/` and `/sp/` exploit strings.

2. **Detect Hidden IIS Backdoors via BadIIS Modules**
   Active campaigns exploiting SQL injection are utilizing the 'BadIIS' module for stealthy persistence on Microsoft IIS web servers. Run the command `appcmd.exe list modules` on all IIS web hosts to inventory active server extensions and flag any unsigned, non-standard DLLs. Correlate this audit with system event logs to detect Windows Defender exclusions configured by attackers to hide high-privilege cryptomining tools and exfiltration scripts.

## Recommandations Stratégiques

1. **Déployer des clés matérielles FIDO2 pour les dépôts**
   Les récentes attaques contre la chaîne d'approvisionnement logicielle, comme le compromis du paquet npm Jscrambler et la fuite GitHub de la CISA, démontrent que les attaquants ciblent activement les environnements de développement. Les organisations doivent abandonner le MFA par SMS ou application au profit de clés matérielles FIDO2/WebAuthn (ex: YubiKey 5) pour sécuriser les comptes de développeurs et les pipelines CI/CD. Ce contrôle direct empêche le vol d'identifiants par des infostealers comme LummaC2 et protège le code source.

2. **Restreindre le middleware d'IA à l'hôte local**
   Avec l'émergence de campagnes de scan ciblant les serveurs Model Context Protocol (MCP) et les secrets Cursor/Claude, l'exposition publique des couches d'orchestration IA représente une faille critique. Forcez toutes les instances locales de serveurs MCP et d'agents IA à écouter exclusivement sur l'interface locale (127.0.0.1) ou restreignez l'accès via des reverse proxies authentifiés utilisant le mTLS. Cette mesure bloque les exécutions JSON-RPC à distance non autorisées et les injections de prompt de type Ghostcommit.

## Recommandations Opérationnelles

1. **Auditer et éteindre les contrôleurs Progress ShareFile**
   Suite à la directive urgente de Progress Software concernant une menace active ciblant les contrôleurs de zone de stockage (liée aux CVE-2023-24489 et CVE-2026-50656), les administrateurs doivent éteindre immédiatement les serveurs Windows concernés. Exécutez la commande PowerShell `Stop-Service -Name 'CitrixShareFileStorageZones'` pour arrêter le service de manière sécurisée et isoler les serveurs au niveau du pare-feu. Ne redémarrez pas les instances avant d'avoir appliqué les correctifs officiels et inspecté les journaux IIS à la recherche de requêtes d'exploitation.

2. **Détecter les portes dérobées IIS via BadIIS**
   Des campagnes actives exploitant des injections SQL installent le module malveillant 'BadIIS' pour maintenir une persistance discrète sur les serveurs web Microsoft IIS. Exécutez la commande `appcmd.exe list modules` sur l'ensemble de vos serveurs IIS pour inventorier les extensions actives et signaler toute DLL non signée ou anormale. Corrélez cet audit avec les journaux d'événements système pour détecter d'éventuelles règles d'exclusion ajoutées dans Windows Defender par les attaquants.
