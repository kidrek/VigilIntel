## Executive Summary

July 2026 marked a pivotal shift in the threat landscape with the operationalization of autonomous Agentic Threat Actors (ATAs) like JADEPUFFER, executing end-to-end database encryption and credential theft via LLM environments. In parallel, software supply chains were heavily compromised through poisoned npm/PyPI registries and hijacked DevOps utilities (TeamPCP, Jscrambler), targeting developer workflows to harvest cloud keys. At the network boundary, threat actors rapidly weaponized high-severity exploits in Microsoft SharePoint, Citrix NetScaler, SonicWall SMA, and Oracle Payments. State-sponsored espionage remained intense, featuring stealthy Chinese kernel-level rootkits (Daxin) in Taiwan and Russian campaigns (FSB Center 16, Turla) targeting critical European router infrastructure and diplomatic networks.

### PART 2 — Geographic Analysis

### Russia

Russian state-sponsored actors (such as FSB Center 16, Turla, and GRU Unit 29155) have intensified highly targeted cyberespionage, router configuration theft, and network sabotage campaigns across Europe and North America. They combine legacy exploit weaponization with stealthy custom implants (BusySnake, BadIIS). Concurrently, Russian cybercriminal syndicates operate with domestic impunity, developing highly structured double-extortion ransomware and 'ClickFix' social engineering networks.

### China

Chinese cyber-espionage operations (such as Warp Panda, Mustang Panda, and the resurrected Daxin group) focus on high-technology, manufacturing, and governmental targets. Their techniques involve compromising edge firewalls and utilizing advanced, kernel-level passive rootkits (Stupig backdoor) to systematically intercept and reroute targeted network traffic within East Asia and Western cloud infrastructures.

### North Korea

North Korean state-linked groups (such as Lazarus) prioritize software supply chain compromises, particularly poisoning npm/PyPI registries, to exfiltrate source code and developer cloud credentials. They also deploy highly sophisticated social engineering and employment fraud schemes to gain remote developer access within Western enterprises.

## Trends & Weak Signals

**Emerging techniques:**
- Autonomous LLM-driven agentic attacks (Agentic Threat Actors) bypassing human operators
- Abuse of Model Context Protocol (MCP) servers for autonomous AI tool hijacking and remote prompt execution
- Blockchain fallback C2 utilizing Polygon smart contracts for resilient, censorship-proof infrastructure resolution
- Adversary-in-the-Middle (AiTM) OAuth session token hijacking bypassing hardware-less MFA controls
- ClickFix social engineering abusing system clipboards and forcing manual PowerShell terminal execution

**Declining techniques:**
- Traditional installation scripts in npm (postinstall hooks) due to security tools enforcing --ignore-scripts policies
- Static file-based malware droppers lacking in-memory execution, ETW evasion, or dynamic code modification
- Broad, unauthenticated database access scanning on standard ports without bypassing MFA layers

**Geopolitical context:** Structural dependency of Western enterprises on US payment networks and hyper-scaler cloud nodes creates sovereignty risks, combined with state-sponsored campaigns from China (Mustang Panda, Daxin rootkits) targeting East Asian infrastructure. Concurrently, European nations face persistent hybrid warfare from Russia (FSB Center 16, Turla), driving public attribute statements, coordinated economic asset freezes, and the mobilization of defensive resources like the EU Cybersecurity Reserve.

## Strategic Recommendations

1. **Enforce FIDO2 Hardware MFA for Accounts**
   Traditional SMS/TOTP MFA is systematically bypassed by AiTM proxies like EvilTokens and OAuth token-hijacking campaigns (observed in the Vercel/ShinyHunters incidents). Transition all administrative, developer, and high-privilege corporate accounts to FIDO2 WebAuthn hardware keys (such as YubiKey 5 or Google Titan). Implement strict Entra ID or AWS Conditional Access policies to enforce hardware-bound device verification.

2. **Establish Continuous Identity Threat Detection (ITDR)**
   State-sponsored actors (Warp Panda, LSHIY) target Azure CLI, SaaS portals, and cloud configurations using stolen credentials and rogue OAuth integrations. Deploy specialized ITDR solutions to continuously audit third-party SaaS API delegations and non-human identity (NHI) privileges. Prohibit non-administrative users from registering external enterprise applications or granting wild-card consents.

## Operational Recommendations

1. **Block Public Exposure of AI Orchestrators**
   The autonomous agent JADEPUFFER exploited missing authentication in Langflow (CVE-2025-3248) to gain initial access and execute unauthorized SQL encryption. Isolate all AI orchestrators, Model Context Protocol (MCP) servers, and developer notebook environments behind VPN/ZTNA gateways. Force local MCP instances to bind strictly to the loopback interface (127.0.0.1) and run AI execution agents in isolated, rootless containers.

2. **Apply Emergency Patches for Citrix NetScaler**
   Active exploitation of Citrix NetScaler memory disclosure (CVE-2025-5777) and remote code execution vulnerabilities allows unauthenticated session hijacking. Apply vendor patches to all NetScaler ADC and Gateway appliances immediately. Post-update, terminate all active AAA sessions on the devices, force full administrative credential rotations, and scan logs for anomalous SAML assertions.

3. **Disable Legacy Cisco Smart Install Service**
   Russian state espionage (FSB Center 16) actively exploits legacy router protocol vulnerabilities (CVE-2018-0171) to extract enterprise configurations and establish persistence. Execute the administrative command 'no vstack' on all enterprise Cisco switches to completely disable the Smart Install service. Restrict external access to SNMP interfaces (ports 161/162) using ACLs and mandate SNMPv3 with encryption.

### PARTIE 2 — Analyse Géographique

### Russie

Les acteurs étatiques russes (tels que le Center 16 du FSB, Turla et l'Unité 29155 du GRU) ont intensifié leurs cyber-espionnages, vols de configurations de routeurs et sabotages réseau en Europe et Amérique du Nord. Ils combinent la militarisation d'exploits avec des implants furtifs (BusySnake, BadIIS). En parallèle, les syndicats cybercriminels russes opèrent avec impunité, développant des rançongiciels à double extorsion et des réseaux d'ingénierie sociale de type 'ClickFix'.

### Chine

Les opérations d'espionnage cyber chinois (Warp Panda, Mustang Panda, et le groupe Daxin) se concentrent sur la haute technologie, l'industrie et les cibles gouvernementales. Leurs techniques impliquent la compromission de pare-feux périmétriques et l'utilisation de rootkits passifs au niveau du noyau (backdoor Stupig) pour intercepter et rediriger furtivement le trafic réseau.

### North Korea

Les groupes liés à la Corée du Nord (comme Lazarus) privilégient les compromissions de supply chain logicielle, en empoisonnant les registres npm/PyPI, pour exfiltrer du code source et des clés cloud. Ils déploient également des fraudes sophistiquées à l'emploi pour obtenir des accès développeurs dans les entreprises occidentales.

## Tendances & Signaux Faibles

**Techniques émergentes :**
- Attaques agentiques autonomes pilotées par LLM sans intervention d'opérateurs humains
- Abus de serveurs Model Context Protocol (MCP) pour le détournement d'outils d'IA autonomes et l'exécution de prompts à distance
- C2 avec repli sur blockchain exploitant les contrats intelligents Polygon pour une résolution d'infrastructure résiliente
- Détournement de jetons de session OAuth par proxy inverse (AiTM) contournant le MFA classique sans clé matérielle
- Ingénierie sociale 'ClickFix' détournant le presse-papiers et incitant à l'exécution manuelle de commandes PowerShell

**Techniques en déclin :**
- Scripts d'installation npm traditionnels (hooks postinstall) en raison de la généralisation des politiques --ignore-scripts
- Droppers de fichiers statiques sans exécution en mémoire, évasion ETW ou modification dynamique de code
- Scans bruts non authentifiés de ports de bases de données standard sans mécanismes de contournement du MFA

**Contexte géopolitique :** La dépendance structurelle des entreprises occidentales envers les systèmes de paiement américains et les hyper-scalers crée des risques majeurs de souveraineté, accentués par les campagnes chinoises (Mustang Panda, Daxin) ciblant l'Asie de l'Est. En parallèle, les pays européens font face à une guerre hybride russe persistante (FSB Center 16, Turla), provoquant des attributions publiques, des gels d'avoirs économiques et la mobilisation de la réserve de cybersécurité de l'UE.

## Recommandations Stratégiques

1. **Imposer le MFA matériel FIDO2 d'entreprise**
   Le MFA traditionnel (SMS/TOTP) est systématiquement contourné par les proxies AiTM comme EvilTokens et le vol de jetons OAuth (incidents Vercel/ShinyHunters). Migrez l'ensemble des comptes administrateurs, développeurs et sensibles vers des clés physiques FIDO2 WebAuthn (YubiKey 5, Google Titan). Configurez des accès conditionnels Entra ID ou AWS pour exiger une validation matérielle.

2. **Déployer une détection d'identité continue (ITDR)**
   Des attaquants étatiques (Warp Panda, LSHIY) ciblent les accès Azure CLI, portails SaaS et configurations cloud via des jetons volés et des applications OAuth détournées. Déployez une solution ITDR pour auditer en continu les intégrations SaaS tierces et les privilèges d'identités non-humaines (NHI). Interdisez aux utilisateurs non-administrateurs d'enregistrer des applications d'entreprise.

## Recommandations Opérationnelles

1. **Isoler les orchestrateurs d'IA du public**
   L'agent autonome JADEPUFFER a exploité l'absence d'authentification dans Langflow (CVE-2025-3248) pour chiffrer des bases de données. Isolez tous les orchestrateurs d'IA, serveurs MCP et environnements de notebook derrière des passerelles VPN/ZTNA. Forcez les serveurs MCP à écouter sur l'interface locale (127.0.0.1) et exécutez les agents d'IA dans des conteneurs isolés sans droits root.

2. **Déployer les correctifs Citrix NetScaler d'urgence**
   L'exploitation des failles de divulgation mémoire de Citrix NetScaler (CVE-2025-5777) permet le détournement de sessions sans authentification. Appliquez immédiatement les correctifs officiels sur vos équipements ADC et Gateway. Révoquez toutes les sessions VPN actives après la mise à jour, forcez la rotation des mots de passe d'administration et inspectez les journaux SAML.

3. **Désactiver le service Cisco Smart Install**
   L'espionnage russe (FSB Center 16) exploite activement des failles protocolaires obsolètes sur routeurs (CVE-2018-0171) pour exfiltrer les configurations d'infrastructure. Exécutez la commande d'administration 'no vstack' sur l'ensemble de vos commutateurs Cisco. Restreignez l'accès à SNMP (ports 161/162) via des ACL strictes et imposez l'usage de SNMPv3 chiffré.
