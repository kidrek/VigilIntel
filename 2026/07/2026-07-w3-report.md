## Strategic Recommendations

1. **Enforce FIDO2 Hardware MFA globally**
   Adversary-in-the-Middle (AiTM) phishing tools, particularly the emerging Kratos PhaaS and EvilProxy platforms, easily bypass standard SMS and TOTP MFA. Organizations must transition to FIDO2/WebAuthn hardware keys (like YubiKey 5 Series or Google Titan) for all enterprise Microsoft 365 and Entra ID accounts. This completely neutralizes the threat of token theft and session hijacking observed in recent campaigns.

2. **Isolate CI/CD Build Runners completely**
   TeamPCP is actively exploiting permissive GitHub Actions workflow configurations (like pull_request_target) to exfiltrate OIDC tokens and poison npm packages like AsyncAPI. Security teams must restrict workflow permissions, enforce SLSA-verifier policies, and utilize ephemeral, network-isolated runners to prevent credential exfiltration. This mitigates the risk of supply chain contamination before deployment to production environments.

## Operational Recommendations

1. **Update SharePoint and audit local installations**
   Active exploitation of SharePoint RCE flaws (CVE-2026-58644 and CVE-2026-56164) is being leveraged to compromise internal corporate directories. Administrators must immediately apply the Microsoft July 2026 cumulative updates on all SharePoint servers. Active health checks on AMSI integration should be validated using Get-SharePointDiagnostics to verify active defense posture.

2. **Harden enterprise routers against SNMP scans**
   Russian state actor FSB Center 16 is actively scanning for exposed SNMPv1/v2 configurations to exfiltrate configuration files (CVE-2018-0171). Network administrators must disable Cisco Smart Install using the no vstack command and restrict SNMP access to local subnet ranges using ACLs. Ensure all community strings are rotated away from defaults to prevent unauthorized configuration dumping.

Les organisations doivent imposer l'usage de clés matérielles FIDO2 (YubiKey 5, Google Titan) sur tous les comptes M365 et Entra ID. Cela neutralise définitivement le vol de jetons de session observé dans les récentes campagnes.

2. **Isoler totalement les runners CI/CD**
   TeamPCP exploite activement les configurations permissives de GitHub Actions (telles que pull_request_target) pour exfiltrer des jetons OIDC et empoisonner les paquets npm d'AsyncAPI. Les équipes doivent restreindre les permissions des workflows, valider la provenance SLSA et isoler les runners du réseau externe. Cela élimine les risques de contamination de la chaîne d'approvisionnement logicielle.

## Recommandations Opérationnelles

1. **Mettre à jour SharePoint Server immédiatement**
   L'exploitation active des vulnérabilités SharePoint (CVE-2026-58644 et CVE-2026-56164) permet des exécutions de code à distance (RCE) critiques. Les administrateurs doivent déployer d'urgence les correctifs Microsoft de juillet 2026 sur tous les serveurs exposés. Validez le fonctionnement d'AMSI avec Get-SharePointDiagnostics pour confirmer l'état défensif global.

2. **Sécuriser les routeurs contre l'espionnage SNMP**
   Le groupe étatique russe FSB Center 16 cible de vieux protocoles SNMP (CVE-2018-0171) pour voler les configurations de routeurs d'entreprise Cisco. Désactivez immédiatement Smart Install avec la commande no vstack sur tous les équipements affectés. Limitez l'accès SNMP aux seuls réseaux locaux internes via des règles ACL strictes.
