## Strategic Recommendations

1. **Enforce FIDO2 Hardware MFA for Developer Accounts**
   Traditional SMS or TOTP-based MFA is systematically bypassed by modern Adversary-in-the-Middle (AiTM) phishing platforms such as Kratos and Tycoon observed this period. Mandate the deployment of FIDO2 WebAuthn hardware security keys (e.g., YubiKey 5, Google Titan) for all developer, DevOps, and cloud administrator accounts. This strictly mitigates the risk of session hijacking and credential harvesting in Microsoft 365 and AWS environments.

2. **Audit and Limit Cloud Service Account Privileges**
   Attackers are actively exploiting over-privileged non-human identities (NHI) and GCP default service accounts via Local File Inclusion (LFI) to escalate privileges to Project Editor. Transition all serverless architectures and cloud functions (GCP Cloud Run, AWS Lambda) away from default configurations. Implement custom service accounts governed by strict IAM Least Privilege policies and short-lived tokens to mitigate tenant takeover risks.

## Operational Recommendations

1. **Harden Developer Systems Against npm Supply Chain Compromises**
   TeamPCP is actively poisoning npm dependencies like AsyncAPI executing code at import-time. Configure developers' environments to run 'npm install --ignore-scripts' by default, and enforce package locking via checksum verification in CI/CD pipelines. Integrate the open-source slsa-verifier tool to validate build provenance and block un-signed open-source libraries.

2. **Disable Legacy Cisco Services to Block State Espionage**
   Russian state-sponsored actors (FSB Center 16) are actively exploiting legacy routing flaws (CVE-2018-0171 and CVE-2008-4128) to exfiltrate core configuration files. Immediately execute the command 'no vstack' on all enterprise Cisco IOS and IOS-XE switches to disable the Smart Install service. Restrict network access to SNMP interfaces (ports 161/162) using strict ACLs, and mandate SNMPv3 with encryption.

Cela est illustré par l'attribution publique par la France d'opérations d'espionnage à l'unité 61240 du FSB russe, et par la révélation des liens structurels du gestionnaire de mots de passe Passwork avec le FSB russe. En parallèle, les tensions restent vives en Asie de l'Est avec le déploiement de rootkits chinois dissimulés à Taïwan.

## Recommandations Stratégiques

1. **Imposer l'authentification MFA matérielle FIDO2 des développeurs**
   Le MFA traditionnel par SMS ou TOTP est systématiquement contourné par les plateformes de phishing AiTM telles que Kratos et Tycoon observées cette semaine. Déployez des clés de sécurité physiques FIDO2/WebAuthn (YubiKey 5, Google Titan) pour tous les comptes d'administration cloud et de développeurs. Cela élimine le risque de vol de session et de contournement MFA sur Microsoft 365 et Azure.

2. **Auditer et restreindre les privilèges des comptes de services**
   Les attaquants exploitent les privilèges excessifs d'identités non-humaines (NHI) et de comptes par défaut (GCP Default Service Account) via des failles LFI pour s'élever au rang d'éditeur. Configurez des comptes de services personnalisés appliquant le moindre privilège et l'usage de jetons de session éphémères pour vos fonctions serverless (GCP Cloud Run, AWS Lambda) afin d'isoler l'infrastructure.

## Recommandations Opérationnelles

1. **Sécuriser les postes de développements face aux attaques npm**
   L'acteur TeamPCP empoisonne activement des paquets npm comme AsyncAPI pour exécuter du code malveillant au chargement du module. Configurez l'environnement de développement pour forcer la commande 'npm install --ignore-scripts' par défaut et imposez la vérification des sommes de contrôle dans la CI/CD. Intégrez l'utilitaire slsa-verifier pour certifier la provenance des dépendances.

2. **Désactiver les protocoles Cisco obsolètes contre l'espionnage russe**
   Les groupes étatiques russes (FSB Center 16) exploitent activement des vulnérabilités de routeurs (CVE-2018-0171 et CVE-2008-4128) pour voler les configurations réseau. Exécutez immédiatement la commande d'administration 'no vstack' sur l'ensemble de vos commutateurs Cisco IOS/IOS-XE. Restreignez l'accès à SNMP (ports 161/162) par des ACL strictes et imposez SNMPv3 chiffré.
