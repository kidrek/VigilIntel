## Executive Summary

This week marks a paradigm shift in threat landscapes with the documentation of the first fully autonomous, AI-driven ransomware campaign by JADEPUFFER, exploiting Langflow (CVE-2025-3248). Concurrently, corporate environments suffered severe identity-based and supply chain intrusions. ## Trends & Weak Signals

**Emerging techniques:**

- Phantom Squatting (hijacking AI-hallucinated domain names in developer code)
- Device Code Flow Abuses (EvilTokens/ARToken MFA bypass via RFC 8628 abuse)

**Declining techniques:**
- Traditional SMS or TOTP MFA protections (routinely bypassed by modern AiTM proxies)
- Static signature-based web shell detection (now frequently bypassing checks via backup file hosting)

**Geopolitical context:** Cyber operations are closely mirroring physical and economic conflicts. China-aligned threat groups focus heavily on European diplomatic targets (OTAN/Greece) and supply chains, while Russian groups continue targeting Western infrastructure and communication channels (Tchap, Signal). ## Strategic Recommendations

1. **Enforce FIDO2 Hardware MFA for Developer Accounts**
   Traditional SMS/TOTP MFA is routinely bypassed by EvilTokens Device Code phishing and adversary-in-the-middle (AiTM) proxied attacks (seen in recent ShinyHunters campaigns). Deploy FIDO2/WebAuthn hardware keys (e.g., YubiKey 5, Google Titan) for all developer and administrative accounts with access to cloud infrastructure and CI/CD pipelines. Revoke all active session tokens immediately upon rollout to evict dormant hijackers.

2. **Establish an Enterprise Post-Quantum Cryptography Roadmap**
   Under ANSSI-2026-NQE and US EO 14412/14413 directives, organizations must transition to Post-Quantum Cryptography (PQC) before 2030 to mitigate 'harvest now, decrypt later' vectors. Initiate an immediate cryptographic discovery phase to inventory all legacy PKI, certificates, and TLS configurations. Partner with vendors to deploy hybrid classical-PQC algorithms compliant with NIST FIPS 203/204/205 standards.

## Operational Recommendations

1. **Disable OAuth ROPC Protocol to Stop LSHIY Campaign**
   The LSHIY password spray campaign actively exploits the Resource Owner Password Credentials (ROPC) protocol to bypass MFA on Azure CLI. Operationally disable ROPC globally inside Microsoft Entra ID Conditional Access policies. Create real-time alerts in your SIEM monitoring for sign-in failures matching Microsoft Entra ID error code '50126'.

2. **Isolate Langflow Instances to Mitigate JADEPUFFER Ransomware**
   JADEPUFFER autonomously exploits unauthenticated Langflow APIs (CVE-2025-3248) to drop SQL-encrypting scripts. Immediately restrict exposure of all local or containerized Langflow, Flowise, or similar LLM orchestration instances. Implement strict egress network filtering on servers running AI workloads, blocking connections to untrusted VPS hosting providers.

## Recommandations Stratégiques

1. **Imposer le MFA matériel FIDO2 pour les développeurs**
   Le MFA traditionnel par SMS/TOTP est contourné par le phishing Device Code EvilTokens et les attaques par proxy inverse (vus dans les récentes campagnes de ShinyHunters). Déployez des clés matérielles FIDO2/WebAuthn (YubiKey 5, Google Titan) pour tous les comptes d'administration et de développement ayant accès aux infrastructures cloud et aux pipelines CI/CD. Révoquez toutes les sessions actives lors du déploiement.

2. **Établir une feuille de route d'entreprise pour la transition quantique**
   Conformément à la directive ANSSI-2026-NQE et à l'EO 14412/14413 US, les organisations doivent migrer vers la cryptographie post-quantique (PQC) d'ici 2030 pour contrer la menace 'harvest now, decrypt later'. Initiez une phase de découverte pour inventorier l'ensemble des PKI, certificats et configurations TLS. Collaborez avec les éditeurs pour déployer des algorithmes hybrides conformes au NIST FIPS 203/204/205.

## Recommandations Opérationnelles

1. **Désactiver le protocole OAuth ROPC contre la campagne LSHIY**
   La campagne de password spraying LSHIY exploite activement le protocole Resource Owner Password Credentials (ROPC) pour contourner le MFA sur Azure CLI. Désactivez globalement ROPC dans les règles d'Accès Conditionnel de Microsoft Entra ID. Créez des alertes temps réel dans votre SIEM pour surveiller les échecs de connexion associés au code d'erreur Entra ID '50126'.

2. **Isoler les instances Langflow contre le ransomware JADEPUFFER**
   JADEPUFFER exploite de manière autonome les API non authentifiées de Langflow (CVE-2025-3248) pour chiffrer les bases SQL. Restreignez immédiatement l'exposition de toutes les instances Langflow, Flowise ou orchestrateurs de LLM similaires. Mettez en œuvre un filtrage strict des flux sortants sur les serveurs exécutant des charges IA pour bloquer les connexions vers les VPS non autorisés.
