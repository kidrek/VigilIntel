## Executive Summary

Week 1 of July 2026 is characterized by a significant surge in identity-based compromises and zero-day exploitation of critical business platforms. ## Trends & Weak Signals

**Emerging techniques:**
- AI-Agentic runtime threats utilizing LLMs like Claude for automated command execution and prompt-driven exfiltration.
- EvilTokens and Device Code phishing to bypass multi-factor authentication (MFA) and claim long-lived cloud session tokens.
- Leveraging trusted redirection services such as Google Maps API parameters (maps.google.co.kr) to defeat secure web gateways.

**Declining techniques:**
- Simple SMS-based OTP configurations, heavily abandoned by administrators in favor of application-based MFA due to rapid SIM-swapping.
- Pure file-based malware droppers, being replaced by API-based post-exploitation and living-off-the-cloud techniques.

**Geopolitical context:** Strong focus on European technological sovereignty. ## Strategic Recommendations

1. **Enforce Phishing-Resistant FIDO2 Hardware MFA**
   Traditional push-based and SMS MFA are easily bypassed by EvilTokens Device Code phishing attacks, as demonstrated by groups like Scattered Spider and ShinyHunters. Deploy FIDO2/WebAuthn hardware tokens (e.g., YubiKey 5 Series) specifically targeting all administrative cloud endpoints (Entra ID, Okta) and engineering teams. Enforce physical keys as the sole authentication method for high-risk profiles and revoke all concurrent sessions upon rollout.

2. **Establish emergency ERP and Gateway SLA**
   Zero-day attacks on Oracle PeopleSoft (CVE-2026-35273) and remote OIDC validation failures (CVE-2026-48558) represent immediate entry points for ransomware operators like Stormous. Define a mandatory 24-hour patching SLA for CISA KEV catalog vulnerabilities on external-facing environments. Systematically segment internal ERP and HCM structures into isolated networks, inaccessible from the public web without dedicated VPN tunnels.

## Operational Recommendations

1. **Audit M365 Service Principal privileges**
   To prevent standard accounts from elevating to Global Admin within minutes, immediately audit all Enterprise Application credential changes. Execute `Get-MgServicePrincipal` via PowerShell to identify newly added secrets or certificates. Restrict and alert on highly dangerous API permissions such as RoleManagement.ReadWrite.Directory and Directory.ReadWrite.All, ensuring no standard user holds ownership of critical Service Principals.

2. **Harden Oracle PeopleSoft web application servers**
   Isolate all instances of PeopleSoft HCM/FSCM and inspect reverse-proxy logs for unusual SOAP/JSP requests targeting `/psc/*` or `/signon/`. Execute automated integrity checks across the `PS_HOME` directory structure to hunt for newly spawned Java web shells. Rotate all service account credentials associated with database connections and verify Oracle's latest CPU patch level.

## Recommandations Stratégiques

1. **Déployer l'authentification FIDO2 résistante au phishing**
   Les méthodes MFA traditionnelles par push et SMS sont de plus en plus contournées par les attaques de type EvilTokens/Device Code, comme l'illustrent ShinyHunters et d'autres acteurs. Déployez des clés matérielles FIDO2/WebAuthn (ex. YubiKey 5) pour les administrateurs cloud (Entra ID, Okta) et équipes techniques. Imposez ces clés physiques comme seule méthode de validation et révoquez toutes les sessions actives lors du déploiement.

2. **Imposer un SLA d'urgence ERP et passerelles**
   Les zero-days sur Oracle PeopleSoft (CVE-2026-35273) et les défauts de validation OIDC (CVE-2026-48558) sont activement exploités par des rançonneurs comme Stormous. Instaurez un SLA de correction de 24h pour les failles du catalogue KEV du CISA sur le périmètre externe. Segmentez rigoureusement les instances ERP et RH dans des réseaux isolés, invisibles depuis Internet sans tunnel VPN dédié.

## Recommandations Opérationnelles

1. **Auditer les privilèges des Service Principals M365**
   Pour empêcher un compte standard de devenir Global Admin en quelques minutes, auditez immédiatement les modifications de credentials d'applications. Lancez `Get-MgServicePrincipal` en PowerShell pour détecter de nouveaux secrets ou certificats. Restreignez et alertez sur les permissions API critiques telles que RoleManagement.ReadWrite.Directory et Directory.ReadWrite.All.

2. **Sécuriser les serveurs d'applications Oracle PeopleSoft**
   Isolez toutes les instances PeopleSoft HCM/FSCM et inspectez les logs des proxys à la recherche de requêtes SOAP/JSP suspectes vers `/psc/*` ou `/signon/`. Lancez des contrôles d'intégrité sur l'arborescence `PS_HOME` pour chasser d'éventuels web shells Java. Renouvelez tous les mots de passe des comptes de service liés à la base de données.
