## Strategic Recommendations

1. **Enforce FIDO2 Hardware MFA on Edge Management**
   Traditional SMS or app-based push MFA is systematically bypassed by helpdesk social engineering (used by Scattered Spider) and offline credential reuse campaigns (FortiBleed). Deploy FIDO2/WebAuthn hardware tokens (such as YubiKey 5 or Google Titan) as the sole verification factor for edge appliance administration, VPN access points, and Active Directory administrative accounts. Revoke all active sessions upon deployment.

2. **Implement Rigorous SaaS Third-Party Risk Management**
   Supply chain compromises via third-party SaaS tools like TinyPulse and Klue continue to expose internal credentials and customer support cases. Mandate annual SOC 2 Type II audits, demand strict API token expiration limits, and segment internal identities so that a compromise of an external platform cannot escalate to local corporate directory systems.

## Operational Recommendations

1. **Rotate FortiGate Admin Credentials and Disable WAN Management**
   The FortiBleed dataset contains plaintext credentials for 73,932 devices globally. Immediately run an audit of all FortiGate appliances, rotate 100% of local and service administrative passwords, and enforce firewall policies to block administrative access from the public WAN. Restrict management interfaces to isolated VLANs accessed only via secure corporate bastions.

2. **Deploy Browser Extension Blocklists via GPO**
   Campaigns like EvilTokens steal OAuth sessions through malicious browser extensions. Implement a strict administrative allowlist for extensions in Google Chrome and Microsoft Edge using Active Directory Group Policy Objects (GPOs) or MDM profiles (e.g., setting 'ExtensionInstallBlocklist' to '*'). Audit existing endpoints for unauthorized plugins loaded from local directories.

## Recommandations Stratégiques

1. **Imposer le MFA matériel FIDO2 sur le périmètre**
   Le MFA traditionnel par SMS ou push est contourné par ingénierie sociale de helpdesks (Scattered Spider) ou réutilisation d'identifiants (FortiBleed). Déployez des clés matérielles FIDO2/WebAuthn (YubiKey 5, Google Titan) comme seul facteur d'administration pour les pare-feux, serveurs VPN et comptes AD privilégiés. Révoquez toutes les sessions actives dès le déploiement.

2. **Mettre en œuvre un contrôle rigoureux du risque SaaS tiers**
   Les compromissions de la chaîne d'approvisionnement via des applications SaaS comme TinyPulse ou Klue exposent les identifiants internes. Exigez des audits SOC 2 Type II annuels, imposez des durées d'expiration courtes sur les jetons d'API et segmentez logiquement les identités pour empêcher qu'un accès tiers compromis ne pivote sur l'annuaire interne.

## Recommandations Opérationnelles

1. **Changer les mots de passe FortiGate et désactiver l'accès WAN**
   Le dataset FortiBleed contient des identifiants en clair de 73 932 pare-feux. Auditez immédiatement tous les équipements FortiGate, renouvelez 100% des mots de passe d'administration locale et de service, et appliquez des règles d'accès strictes pour interdire le management depuis le WAN. Restreignez l'accès à un VLAN d'administration isolé accessible via bastion.

2. **Bloquer les extensions de navigateur non autorisées via GPO**
   Des campagnes comme EvilTokens volent les sessions OAuth via des extensions malveillantes. Déployez une allowlist stricte pour Google Chrome et Microsoft Edge en utilisant des objets de stratégie de groupe (GPO) ou profils MDM (ex: configurer 'ExtensionInstallBlocklist' à '*'). Auditez les postes à la recherche d'extensions chargées localement.
