## Trends & Weak Signals

**Emerging techniques:**
- Prompt injection attacks in enterprise generative AI (LLM) models to override medical diagnostics and exfiltrate records.
- eBPF-based kernel rootkits compiled in Rust embedded within software repository installs (Arch Linux AUR).
- Abusing the Model Context Protocol (MCP) to access sensitive data stores (Splunk, CrowdStrike) via unauthenticated AI integration endpoints.

**Declining techniques:**
- Traditional file-encryption-only ransomware is being replaced by double extortion and pure data exfiltration without deployment of lockers.
- Static code scanners for dependency security as they fail to detect dynamically injected `.pth` loaders and build runtime cache poisoning.

**Geopolitical context:** Tensions between Western allies and Eastern syndicates continue to weaponize cyber tactics. Russian economy-wide militarization fuels safe harbors for RaaS operations, while Chinese operations focus on long-term pre-positioning in critical infrastructures. Meanwhile, Iran is increasingly utilizing targeted proxy fronts (like Handala) to perform highly visible intrusions on Western municipal utility infrastructures.

## Strategic Recommendations

1. **Enforce FIDO2 Hardware MFA for Developer and ERP Accounts**
   Traditional SMS/TOTP MFA and session tokens are consistently bypassed by callback phishing (UNC3753) and session hijacking (Miasma worm). Deploy FIDO2/WebAuthn hardware keys (YubiKey 5, Google Titan) for all accounts with access to cloud infrastructure, SaaS platforms, and enterprise ERPs like Oracle PeopleSoft. Establish continuous access evaluation (CAEP) to revoke active sessions upon detection of abnormal IP shifts.

2. **Implement Strict Software Supply Chain Integrity Verification**
   Recent supply chain attacks on npm packages (Mini Shai-Hulud) and Arch Linux AUR highlight the fragility of public registries. Organizations must freeze dependency versions, mandate cryptographic commit signing (GPG/SSH), and deploy private artifact repositories (e.g., JFrog Artifactory, Sonatype Nexus). Configure CI/CD pipelines to block raw Git imports and execute builds in hardened, ephemeral, and network-isolated sandboxes.

## Operational Recommendations

1. **Apply Urgent Security Patches for Oracle PeopleSoft and Ivanti Sentry**
   Exploitation of CVE-2026-35273 (Oracle PeopleSoft EMHub RCE) by ShinyHunters and CVE-2026-10520 (Ivanti Sentry Command Injection) is active in the wild. Immediately apply the vendor-provided patches. If patching Oracle PeopleSoft is delayed, disable the PSEMHUB service or block public external access to endpoints like `/PSEMHUB/*` at the WAF level.

2. **Monitor and Restrict Remote Monitoring and Management (RMM) Tools**
   Silent Ransom Group (SRG) and UNC3753 routinely deploy commercial RMM tools (AnyDesk, Zoho Assist, Bomgar) during callback phishing campaigns. Configure AppLocker, WDAC, or endpoint security policies to block unauthorized RMM executables. Audit active parent-child process anomalies, such as `cmd.exe` or `powershell.exe` spawning from support utilities.

## Tendances & Signaux Faibles

**Techniques émergentes :**
- Attaques par injection de prompt dans les modèles d'IA générative (LLM) d'entreprise pour contourner les diagnostics médicaux et exfiltrer des dossiers.
- Rootkits de noyau basés sur eBPF compilés en Rust et intégrés dans les installations de dépôts de logiciels (Arch Linux AUR).
- Abus du protocole de contexte de modèle (MCP) pour accéder à des magasins de données sensibles (Splunk, CrowdStrike) via des points de terminaison d'intégration d'IA non authentifiés.

**Techniques en déclin :**
- Les ransomwares traditionnels limités au chiffrement de fichiers sont remplacés par la double extorsion et l'exfiltration pure de données sans déploiement de chiffreur.
- Les scanners de code statiques pour la sécurité des dépendances, car ils ne parviennent pas à détecter les chargeurs `.pth` injectés de manière dynamique et l'empoisonnement du cache d'exécution de build.

**Contexte géopolitique :** Les tensions entre les alliés occidentaux et les syndicats orientaux continuent d'instrumentaliser les tactiques cyber. La militarisation de l'économie russe favorise des havres de paix pour les opérations RaaS, tandis que les opérations chinoises se concentrent sur le pré-positionnement à long terme dans les infrastructures critiques. En parallèle, l'Iran utilise de plus en plus des fronts de procuration ciblés (comme Handala) pour mener des intrusions très visibles sur les infrastructures de services municipaux occidentaux.

## Recommandations Stratégiques

1. **Imposer l'authentification multifacteur matérielle FIDO2 pour les développeurs et ERP**
   Les authentifications multifacteurs traditionnelles (SMS/TOTP) et les jetons de session sont systématiquement contournés par le callback phishing (UNC3753) et le détournement de session (ver Miasma). Déployez des clés matérielles FIDO2/WebAuthn (YubiKey 5, Google Titan) pour tous les comptes ayant accès à l'infrastructure cloud, aux plateformes SaaS et aux ERP d'entreprise comme Oracle PeopleSoft. Établissez une évaluation continue des accès (CAEP) pour révoquer les sessions actives en cas de détection de changements IP anormaux.

2. **Mettre en œuvre une vérification stricte de l'intégrité de la chaîne d'approvisionnement**
   Les récentes attaques contre la chaîne d'approvisionnement des paquets npm (Mini Shai-Hulud) et d'Arch Linux AUR soulignent la fragilité des registres publics. Les organisations doivent figer les versions de dépendance, imposer la signature cryptographique des commits (GPG/SSH) et déployer des dépôts d'artefacts privés (ex. JFrog Artifactory, Sonatype Nexus). Configurez les pipelines CI/CD pour bloquer les importations Git brutes et exécuter les builds dans des bacs à sable durcis, éphémères et isolés du réseau.

## Recommandations Opérationnelles

1. **Appliquer d'urgence les correctifs pour Oracle PeopleSoft et Ivanti Sentry**
   L'exploitation de CVE-2026-35273 (RCE Oracle PeopleSoft EMHub) par ShinyHunters et CVE-2026-10520 (injection de commandes Ivanti Sentry) est active dans la nature. Appliquez immédiatement les correctifs fournis par les éditeurs. Si l'application du correctif Oracle PeopleSoft est retardée, désactivez le service PSEMHUB ou bloquez l'accès externe public aux endpoints comme `/PSEMHUB/*` au niveau du WAF.

2. **Surveiller et restreindre les outils de gestion à distance (RMM)**
   Le Silent Ransom Group (SRG) et UNC3753 déploient régulièrement des outils RMM commerciaux (AnyDesk, Zoho Assist, Bomgar) lors de campagnes de callback phishing. Configurez AppLocker, WDAC ou des politiques de sécurité des terminaux pour bloquer les exécutables RMM non autorisés. Auditez les anomalies de processus parents-enfants actifs, comme `cmd.exe` ou `powershell.exe` générés à partir d'utilitaires de support.
