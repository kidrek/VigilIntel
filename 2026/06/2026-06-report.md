## Threat Level

**🔴🔴🔴🔴🔴 5/5 — Severe**

The consolidated threat level for June 2026 is critical. This evaluation is driven by active, widespread zero-day exploitations (such as PAN-OS CVE-2026-0257, Check Point VPN CVE-2026-50751, and Oracle PeopleSoft CVE-2026-35273) coupled with highly automated software supply chain worms (Miasma and Shai-Hulud) targeting developer environments. ## Executive Summary

June 2026 represented a highly volatile period characterized by three systemic strategic shifts: the industrialization of credential harvesting at the edge, automated software supply chain infection, and advanced local virtualization-based defense evasion. The perimeter security model was deeply shaken by 'FortiBleed', which harvested configurations from tens of thousands of Fortinet gateways, and by critical zero-days in Check Point and Oracle PeopleSoft systems exploited by groups like ShinyHunters. Concurrently, automated developer worm campaigns (Miasma and Shai-Hulud) compromised multiple official cloud repositories and package managers (npm, AUR). Ransomware operations reached high levels of disruption, utilizing 'Bring Your Own Hypervisor' (QEMU execution) to bypass host-based EDR tools. These significant pressures were partially mitigated by coordinated international law enforcement campaigns, notably Operation Endgame Phase 4, which disrupted the SocGholish distribution network.

Russian-affiliated actors remain the most disruptive force in financially-motivated cybercrime and espionage. Ransomware operators (such as LockBit, Qilin, and The Gentlemen) systematically leverage double-extortion tactics and infrastructure zero-days to compromise global networks, while advanced persistent threat groups (such as APT29 and Gamaredon) target European and Ukrainian state structures with custom loaders and credential harvesting campaigns.

Chinese state-sponsored groups focus on persistent long-term pre-positioning and highly automated scale operations. ## Strategic Recommendations

1. **Enforce FIDO2 Hardware MFA on Administrative Accounts**
   Traditional SMS and push-based MFA are systematically bypassed by modern Adversary-in-the-Middle (AiTM) and device code phishing platforms like Kali365, as seen in the Snowflake (Neiman Marcus) and TeamPCP campaigns. Deploy FIDO2/WebAuthn hardware security keys (such as YubiKey 5 Series) across all enterprise cloud consoles, Entra ID tenants, and remote access systems. This fundamentally mitigates session hijacking risks by cryptographically binding the authentication process to the specific origin domain.

2. **Segment and Isolate OT/IT Administrative Networks**
   Threat groups like Handala (Void Manticore) and Velvet Ant actively target public-facing administrative portals to pivot into critical operational technologies (OT), as demonstrated in the California Water Service breach. Implement strict network micro-segmentation, zero-trust gateways, and automated access control lists (ACLs) to decouple corporate IT billing systems from industrial environments. Mandate dedicated jump hosts with hardware-token-authenticated VPNs for all OT configuration adjustments.

3. **Secure CI/CD Workflows Against Supply Chain Worms**
   Threat actors such as TeamPCP exploit loose pull request triggers to deploy worms like Miasma and Shai-Hulud, compromising GitHub Actions developer runners and extracting cloud secrets. Implement strict branch protection rules, require independent peer reviews for external repository contributors, and restrict runner permissions using the 'permissions: read-all' directive in YAML configurations. Pin all pipeline GitHub Actions to immutable, audited commit SHA-256 hashes instead of mutable semantic version tags.

## Operational Recommendations

1. **Upgrade FortiGate Appliances and Force Password Rotation**
   The global 'FortiBleed' credential harvesting campaign has successfully extracted SHA-256 administrative password hashes from over 73,000 Fortinet VPN gateways. Immediately upgrade vulnerable FortiGate physical and virtual appliances to FortiOS versions 7.2.11+, 7.4.8+, or 7.6.1+ to transition the storage mechanism to PBKDF2 hashing. Following the software update, force a mandatory rotation of all local administrative and user credentials to rewrite the hashes and terminate active OIDC/OAuth sessions.

2. **Patch Oracle PeopleSoft EMHub Vulnerability CVE-2026-35273**
   Active remote exploitation of CVE-2026-35273 by ShinyHunters (UNC6240) has resulted in massive data theft from major academic and financial systems globally. Apply the June 2026 cumulative security update released by Oracle to all PeopleSoft Enterprise PeopleTools environments (versions 8.61 and 8.62). If immediate patching is not feasible, restrict public access to the `/PSEMHUB/*` and `/PSIGW/*` endpoints using a Web Application Firewall (WAF) or local reverse proxy rules.

3. **Restrict Local Hypervisors to Prevent BYOH Ransomware**
   Ransomware operators like Payouts King deploy 'Bring Your Own Hypervisor' (BYOH) tactics by launching Linux-based encryptors within local QEMU virtual machines on compromised Windows hosts to bypass EDR agents. Deploy a domain-wide AppLocker or Windows Defender Application Control (WDAC) policy to explicitly block unauthorized hypervisor binaries (such as `qemu-system-x86_64.exe` or `VirtualBox.exe`). Restrict local virtualization permissions to authorized administrators and monitor system event logs for unauthorized kernel modules.

## Tendances & Signaux Faibles

**Techniques émergentes :**
- Bring Your Own Hypervisor (BYOH) via exécution locale QEMU pour contourner l'EDR
- Intégration de Direct Syscalls par les infostealers (Lumma v17) pour éluder les API hooks
- Compilation WASM de charges utiles de C2 en Go/C# pour contourner les signatures statiques
- Rootkits eBPF ciblant les postes Linux des développeurs pour une persistance indétectable

**Techniques en déclin :**
- Défenses basées sur le hook d'API en mode utilisateur contournées par les syscalls directs
- MFA traditionnel par SMS ou push en raison du phishing automatisé de type Adversary-in-the-Middle (AiTM)
- Commits de développement non signés dans les dépôts d'entreprise

**Contexte géopolitique :** Les tensions géopolitiques façonnent les cyber-opérations en Europe de l'Est, au Moyen-Orient et en Asie de l'Est. La militarisation des contrôles d'exportations technologiques—comme le blocage américain sur l'export de modèles d'IA de pointe—perturbe les capacités défensives mondiales. En parallèle, les acteurs étatiques recourent massivement à des proxys commerciaux (tels que NetNut et Vo1d) et des malwares génériques pour masquer l'attribution et s'ancrer dans les bases de données d'infrastructures d'importance vitale.

## Recommandations Stratégiques

1. **Imposer le MFA matériel FIDO2 sur les comptes privilégiés**
   Les méthodes MFA classiques par SMS ou push sont systématiquement contournées par les outils d'hameçonnage AiTM comme Kali365. Imposez des clés de sécurité matérielles FIDO2/WebAuthn (ex. YubiKey 5) sur toutes les connexions Microsoft Entra ID, VPN et consoles cloud. Cela neutralise définitivement le vol de session et la réutilisation d'identifiants observés récemment.

2. **Segmenter et isoler les réseaux industriels OT/IT**
   Des groupes comme Handala (Void Manticore) et Velvet Ant s'attaquent aux portails administratifs pour s'infiltrer au sein de réseaux de technologies opérationnelles (OT), à l'instar de l'intrusion chez California Water Service. Déployez une micro-segmentation stricte, des architectures de confiance zéro (Zero Trust) et des listes de contrôle d'accès (ACL) automatisées pour isoler la facturation informatique des systèmes de contrôle industriels. Exigez l'usage obligatoire de passerelles de rebond dédiées et de connexions VPN authentifiées par clés physiques.

3. **Sécuriser les workflows CI/CD contre les vers applicatifs**
   Des groupes cybercriminels comme TeamPCP tirent parti de configurations de pull requests permissives pour propager des vers tels que Miasma et Shai-Hulud au sein des runners GitHub Actions. Configurez des règles de protection de branches strictes, exigez une validation manuelle humaine avant l'exécution de pipelines provenant de contributeurs externes, et restreignez les autorisations par défaut des runners via la directive 'permissions: read-all'. Épinglez systématiquement chaque action de build à des empreintes de commit SHA-256 uniques.

## Recommandations Opérationnelles

1. **Mettre à jour FortiGate et forcer la rotation administrateur**
   La vaste campagne 'FortiBleed' a permis de collecter les hashes de mots de passe d'administration de plus de 73 000 passerelles VPN SSL Fortinet à l'échelle globale. Procédez à la mise à niveau immédiate de vos appliances FortiGate vers les versions de FortiOS 7.2.11+, 7.4.8+ ou 7.6.1+ pour activer l'algorithme robuste PBKDF2. À la suite de cette mise à jour, imposez une rotation globale et obligatoire de l'ensemble des identifiants administratifs locaux afin de régénérer la base cryptographique.

2. **Patcher d'urgence la faille Oracle PeopleSoft CVE-2026-35273**
   L'exploitation à distance active de la faille CVE-2026-35273 par ShinyHunters (UNC6240) a provoqué d'importantes fuites de données d'universités et d'institutions financières mondiales. Appliquez sans délai le correctif de sécurité cumulatif de juin 2026 publié par Oracle pour toutes les instances de PeopleSoft Enterprise PeopleTools (versions 8.61 et 8.62). Si le déploiement du correctif doit être différé, configurez vos pare-feux applicatifs (WAF) pour bloquer tout trafic externe vers les répertoires `/PSEMHUB/*` et `/PSIGW/*`.

3. **Bloquer les hyperviseurs locaux pour contrer le BYOH**
   Les groupes de ransomware à l'instar de Payouts King utilisent la technique 'Bring Your Own Hypervisor' (BYOH) en exécutant des outils de chiffrement au sein de machines virtuelles locales QEMU afin de contourner les agents EDR de l'hôte Windows. Déployez une stratégie globale AppLocker ou Windows Defender Application Control (WDAC) pour interdire de manière stricte l'exécution d'exécutables de virtualisation non approuvés (comme `qemu-system-x86_64.exe` ou `VirtualBox.exe`).
