## Executive Summary

June 2026 established a new baseline for systemic threats, defined by a convergence of automated software supply chain poisoning, perimeter zero-day exploitation, and sophisticated endpoint evasion. Threat group TeamPCP dominated the landscape by deploying automated worms (Miasma and Shai-Hulud) to hijack developer environments, compromising dozens of Microsoft Azure-Samples repositories and downstream npm/PyPI pipelines to exfiltrate proprietary source code. Established groups like Qilin and the emerging 'The Gentlemen' operate out of Russian safe-havens, targeting global enterprises with pure data exfiltration and hypervisor-targeted wiper attacks.

Iran-affiliated threat actors focus on highly disruptive, politically motivated campaigns disguised as hacktivism. The group Handala (acting for Void Manticore) targeted US water systems to manufacture claims of physical control and OT sabotage, indicating a continuous effort to execute high-profile psychological and infrastructure operations.

## Strategic Recommendations

1. **Enforce FIDO2 Hardware MFA for Developer Accounts**
   Traditional SMS and TOTP MFA are routinely bypassed by modern session hijacking and automated phishing campaigns (as observed in TeamPCP's credential exfiltrations and the Neiman Marcus Snowflake breach). Mandate the deployment of FIDO2/WebAuthn hardware security keys (e.g., YubiKey 5 Series, Google Titan) for all developer and administrative sessions. Ensure session lifetimes on OIDC and SAML identity providers are restricted to short durations, and force a complete revocation of active tokens upon rollout to eliminate compromised sessions.

2. **Deploy Immutable Air-Gapped Cloud Backups**
   Recent wiper operations and advanced RaaS campaigns (such as Payouts King and The Gentlemen) systematically compromise administrative hypervisor consoles to purge live virtual machines and delete online replication backups. Implement write-once-read-many (WORM) immutable cloud storage buckets for all transaction logs and core systems. Establish a logically isolated, physical backup vault, and require multi-party physical approval (dual control) for any administrative backup modification or deletion request.

## Operational Recommendations

1. **Audit GitHub Actions Workflow Triggers immediately**
   The Shai-Hulud supply chain worm actively exploits misconfigured pull_request_target triggers to execute malicious code within privileged pipeline runners. Use static analysis tools like Semgrep or GitHub Advanced Security to scan all organization repositories for workflows combining pull_request_target with checkout steps of untrusted code. Configure pipeline permissions to restrict the default GITHUB_TOKEN to read-only for all workflow runs initiated from external forks.

2. **Patch Active Edge Gateway Zero-Days immediately**
   Active exploit campaigns are targeting critical remote access gateways, specifically Oracle PeopleSoft EMHub (CVE-2026-35273) and Check Point VPN (CVE-2026-50751). Apply vendor-provided emergency cumulative patches immediately across all public-facing systems. If patching must be delayed, implement temporary web application firewall (WAF) rules to drop requests containing `/PSEMHUB/*` and `/PSIGW/*` URIs, and strictly disable legacy IKEv1 negotiations on all VPN appliances.

## Tendances & Signaux Faibles

**Techniques émergentes :**
- Exécution de charges utiles au sein d'hyperviseurs de machines virtuelles locales QEMU sur hôtes Windows (Bring Your Own Hypervisor)
- Appels système directs vers le noyau (Direct Kernel Syscalls) par les infostealers (Lumma v17) pour aveugler les agents EDR
- Militarisation de vers automatisés (Shai-Hulud, Miasma) ciblant les configurations pull_request_target de GitHub
- Génération par IA générative de templates de phishing dynamiques et de pipelines automatisés de smishing

**Techniques en déclin :**
- Double authentification par SMS ou TOTP sans protection matérielle, systématiquement contournée par vol de sessions et smishing
- Stockage en clair de clés d'API et d'identifiants au sein des dépôts de code
- Injections manuelles de scripts pour l'empoisonnement de dépendances au profit de vers à propagation autonome

**Contexte géopolitique :** Le mois de juin 2026 illustre la convergence accrue entre cyber-espionnage étatique et perturbation industrielle. Les tensions se sont matérialisées par le démantèlement d'infrastructures adverses majeures (comme la plateforme chinoise Outsider Enterprise) par les agences occidentales, tandis que des acteurs basés en Russie répliquaient par des attaques destructrices de type wiper. Parallèlement, le ciblage persistant d'infrastructures critiques (Velvet Ant, Handala) démontre que les tensions géopolitiques se traduisent directement en incidents techniques perturbant les services publics et les circuits d'approvisionnement.

## Recommandations Stratégiques

1. **Imposer l'authentification MFA matérielle FIDO2 pour les développeurs**
   Le MFA traditionnel par SMS ou TOTP est facilement contourné par les campagnes d'infostealers et de piratage de session (campagnes de TeamPCP et compromission Snowflake). Imposez des clés matérielles FIDO2/WebAuthn (ex : YubiKey série 5) pour l'accès aux consoles d'administration, dépôts de code et environnements cloud. Réduisez la durée de vie des sessions et révoquez l'ensemble des jetons actifs lors du déploiement.

2. **Déployer des sauvegardes cloud immuables et isolées**
   Les récentes campagnes de wipers et de rançongiciels ciblent directement les consoles d'administration des hyperviseurs pour détruire les machines virtuelles et supprimer les sauvegardes synchronisées. Implémentez des politiques de stockage cloud immuable de type WORM (Write-Once-Read-Many). Configurez des sauvegardes isolées (air-gapped) et exigez une validation multi-acteurs (double contrôle) pour toute modification de sauvegarde.

## Recommandations Opérationnelles

1. **Auditer immédiatement les déclencheurs de workflows GitHub Actions**
   Le ver Shai-Hulud exploite les déclencheurs pull_request_target mal configurés pour exécuter du code malveillant sur les serveurs de build. Analysez les dépôts à l'aide d'outils d'analyse statique comme Semgrep pour identifier l'usage de pull_request_target combiné à un checkout de code non vérifié, et limitez les droits par défaut du jeton GITHUB_TOKEN en lecture seule.

2. **Patcher immédiatement les failles zero-day des passerelles**
   Des vagues d'attaques exploitent activement les vulnérabilités périphériques critiques d'Oracle PeopleSoft EMHub (CVE-2026-35273) et Check Point VPN (CVE-2026-50751). Appliquez immédiatement les correctifs fournis par les éditeurs. En cas d'impossibilité de redémarrage immédiat, bloquez les URI contenant `/PSEMHUB/*` au niveau du WAF et désactivez les négociations VPN IKEv1 obsolètes.
