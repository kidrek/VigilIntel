## Executive Summary

The consolidated threat landscape of June 2026 shows a highly volatile environment where edge perimeter vulnerability exploitation and software supply chain contamination converge. Cybercriminals and state-sponsored groups are executing complex multi-stage attacks, including 'Bring Your Own Hypervisor' (BYOH) tactics to bypass local EDR detection and eBPF-based persistence in development environments. State-sponsored groups from China (Volt Typhoon, Velvet Ant) and Iran (Handala) maintain persistent espionage and disruptive postures against critical infrastructure, while RaaS and extortion actors (Qilin, ShinyHunters, and The Gentlemen) systematically exploit newly disclosed zero-day vulnerabilities like Oracle PeopleSoft CVE-2026-35273 to extract vast datasets. Concurrently, regulatory bodies globally are tightening compliance, with record fines and stricter disclosure expectations forcing security teams to focus heavily on continuous dependency validation, Zero Trust architectures, and rapid perimeter patching.

### PART 2 — Geographic Analysis

### China

State-sponsored campaigns from China continue to target maritime logistics, managed service providers (MSPs), and government infrastructure. Acting with extreme stealth and often evading host-based EDR, groups such as Velvet Ant (Operation Highland) and Volt Typhoon maintain persistent access on isolated networks. These actors leverage trojanized authentication modules (PAM/OpenSSH) and large-scale botnets (JDY) for long-term strategic positioning, ensuring access that survives standard remediation procedures.

### Russia

Russian state-sponsored groups focus heavily on geopolitical espionage and persistent operations in Ukraine and European diplomatic circles. Concurrently, financially-motivated RaaS operators (such as Qilin and the newly emerged group 'The Gentlemen') operate freely from Russian sanctuaries. These ransomware actors leverage newly discovered network gateway and remote access vulnerabilities to gain administrative footholds, conducting multi-stage extortion and data theft campaigns.

### Iran

Iran-affiliated threat actors (notably Handala, serving as a front for Void Manticore) are increasingly executing disruptive, politically-aligned cyber-physical operations. Rather than seeking purely financial gain, these campaigns target critical public infrastructure (such as California Water Service) to impact municipal management systems and simulate direct physical sabotage, illustrating a rising trend of destructive regional and international hacktivism.

## Strategic Recommendations

1. **Enforce FIDO2 Hardware MFA on All Developer and Admin Accounts**
   Traditional SMS/TOTP and app-based MFA are routinely bypassed by session hijacking and cookie theft, as observed in TeamPCP's cloud credential extractions and ShinyHunters' campaigns. Mandate the use of FIDO2/WebAuthn hardware security keys (e.g., YubiKey 5 Series, Google Titan) for all system developers, cloud administrators, and third-party vendors. Ensure that all existing OIDC/OAuth sessions are programmatically revoked upon deployment to neutralize already compromised tokens.

2. **Segment and Air-Gap Critical Industrial Control Environments**
   Geopolitical targeting of infrastructure, such as Handala's intrusion into California Water Service, exploits weak boundaries between corporate and operational systems. Enforce strict physical air-gapping or advanced logical micro-segmentation between corporate IT services (billing, email) and Operational Technology (OT) zones. Require multi-layered, hardware-authenticated jump boxes for any temporary remote maintenance session.

3. **Implement Strict Controls to Block Unsanctioned Virtualization Software**
   Ransomware groups like Payouts King deploy the 'Bring Your Own Hypervisor' tactic to execute encryption binaries inside local virtual machines (QEMU), completely hiding from host-level EDR. Configure Windows Defender Application Control (WDAC) or AppLocker policies to prevent the execution of unauthorized virtualization binaries (e.g., qemu-system-x86_64.exe, VirtualBox.exe) on endpoints and servers. Ensure file access monitoring rules detect abnormal VM-driven bulk file writes.

## Operational Recommendations

1. **Patch Critical Boundary Edge Vulnerabilities Immediately**
   Emergency edge patches must be prioritized to defeat ongoing campaigns exploiting Check Point VPNs (CVE-2026-50751) and Oracle PeopleSoft EMHub (CVE-2026-35273). Apply the manufacturers' latest security updates immediately, or configure WAF rules to drop traffic targeting the `/PSEMHUB/*` and `/PSIGW/*` endpoints if patching is delayed. Ensure all enterprise VPNs enforce device posture checks before granting access.

2. **Restrict Node-GYP Executions and Enforce Dependency Verification**
   Supply chain infections like the Miasma worm and the Atomic Arch AUR packages leverage floating package versions and malicious install scripts to gain code execution. Configure local package managers to reject unsigned dependencies, block node-gyp compilation triggers by running `npm install --ignore-scripts` in build environments, and force all internal development projects to reference pinned, immutable commit SHA hashes.

3. **Deactivate SVG Document Script Executions on Endpoints**
   Attackers bypass Secure Email Gateways (SEGs) by embedding interactive JavaScript inside vector SVG files to execute dynamic phishing redirects. Deploy a Group Policy Object (GPO) to modify the default Windows handler for `.svg` files, mapping them to static image viewers such as Microsoft Paint or Photos rather than web browsers. Configure your mail transfer agents to block and quarantine any email attachments containing embedded SVG scripts.

## Tendances & Signaux Faibles

**Techniques émergentes :**
- Bring Your Own Hypervisor (Exécution dans QEMU local pour contourner l'EDR)
- Rootkits eBPF insérés directement au sein de paquets AUR et NPM d'outils de développement
- Exfiltration pure de données sans chiffrement pour minimiser l'empreinte de sécurité
- Abus de serveurs de protocoles de contexte de modèles (MCP) pour des mouvements latéraux
- Injections de prompts ciblant les systèmes d'aide à la décision clinique basés sur l'IA

**Techniques en déclin :**
- Double authentification par SMS dénuée de validation matérielle
- Scans réseau classiques non axés sur les dépendances applicatives et le SaaS
- Commits de code non signés et utilisation de tags flottants dans les pipelines CI/CD
- Stockage en clair de clés API et de secrets d'infrastructure dans les dépôts de code

**Contexte géopolitique :** L'activité cyber étatique s'est étroitement mêlée aux enjeux de souveraineté industrielle et de contrôles à l'exportation. ## Recommandations Stratégiques

1. **Imposer l'authentification MFA matérielle FIDO2 sur les comptes sensibles**
   Les MFA traditionnels par SMS/TOTP sont contournés par les attaques de vol de sessions observées ce mois-ci (campagnes TeamPCP et compromissions ShinyHunters). Déployez de manière obligatoire des clés de sécurité matérielles FIDO2/WebAuthn (YubiKey 5, Google Titan) pour tous les comptes développeurs et d'administration. Révoquez toutes les sessions actives OIDC/OAuth lors du déploiement pour neutraliser les jetons volés.

2. **Segmenter et isoler les réseaux industriels OT/IT**
   Le ciblage des infrastructures d'eau par le groupe Handala illustre la fragilité des interconnexions entre systèmes informatiques de gestion et réseaux industriels. Mettez en œuvre un cloisonnement logique strict ou un 'air-gap' physique entre le réseau bureautique IT et l'environnement de contrôle industriel (OT). Exigez des bastions certifiés et une authentification renforcée pour toute opération de maintenance à distance.

3. **Bloquer l'exécution d'hyperviseurs locaux via des règles AppLocker**
   Des groupes comme Payouts King exploitent la tactique 'Bring Your Own Hypervisor' en exécutant leur charge utile au sein de machines virtuelles QEMU locales pour échapper à l'EDR de l'hôte Windows. Implémentez des règles AppLocker ou WDAC restrictives pour interdire le lancement des exécutables de virtualisation (tels que qemu-system-x86_64.exe) sur l'ensemble des postes et serveurs non dédiés.

## Recommandations Opérationnelles

1. **Patcher d'urgence les failles critiques de bordure**
   Des correctifs d'urgence doivent être appliqués immédiatement pour contrer les attaques contre les VPN Check Point (CVE-2026-50751) et Oracle PeopleSoft (CVE-2026-35273). Si le déploiement prend du retard, configurez des filtres WAF pour rejeter le trafic vers `/PSEMHUB/*` et `/PSIGW/*`. Imposez une vérification de la conformité de l'équipement (posture check) avant de valider l'accès aux passerelles VPN.

2. **Restreindre l'exécution de Node-GYP et verrouiller les commits SHA**
   Les contaminations logicielles comme le ver Miasma et les paquets AUR empoisonnés utilisent des versions flottantes et des scripts d'installation cachés. Configurez les gestionnaires de paquets pour rejeter les dépendances non signées, désactivez les scripts d'installation via l'option `--ignore-scripts` lors de `npm install`, et imposez le référencement exclusif par hashs de commits SHA uniques.

3. **Désactiver l'exécution de scripts au sein des fichiers SVG**
   Les attaquants contournent les passerelles de messagerie en insérant du code JavaScript malveillant dans des fichiers vectoriels SVG pour rediriger les utilisateurs. Déployez une GPO sur l'ensemble du parc pour associer l'extension `.svg` à une application d'affichage d'images statique (ex. Paint) au lieu d'un navigateur internet, et configurez la messagerie pour bloquer les SVG contenant des balises scripts.
