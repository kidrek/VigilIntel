## Strategic Recommendations

1. **Enforce FIDO2 Hardware MFA on Edge Management Panels**
   Traditional SMS or app-based push MFA is systematically bypassed by helpdesk social engineering (used by Scattered Spider) and offline credential reuse campaigns (such as FortiBleed). Deploy FIDO2/WebAuthn hardware tokens (such as YubiKey 5 or Google Titan) as the sole verification factor for edge appliance administration, VPN access points, and Active Directory administrative accounts. Revoke all active OIDC/OAuth sessions upon deployment.

2. **Implement AppLocker Rules to Prevent Local Virtualization Abuse**
   Ransomware operators like 'The Gentlemen' and 'Payouts King' are executing Linux-based encryption daemons inside local QEMU virtual machines on compromised Windows hosts to evade EDR. Configure strict AppLocker or Windows Defender Application Control (WDAC) rules to prohibit the execution of virtualization binaries (e.g., qemu-system-x86_64.exe or VirtualBox.exe) on non-authorized endpoints and file servers. This effectively limits lateral movement and prevents silent host-level encryption.

## Operational Recommendations

1. **Rotate FortiGate Credentials and Block WAN Administration**
   The FortiBleed dataset contains compromised administrative credentials for over 73,000 devices globally. Immediately audit all FortiGate and Sophos appliances, rotate 100% of local and service administrative passwords, and enforce firewall policies to block administrative interfaces from the public WAN. Restrict access strictly to isolated management VLANs accessible only via secure corporate bastions.

2. **Enable WHQL Vulnerable Driver Blocklists via GPO**
   EDR-killing tools like GentleKiller rely on the BYOVD (Bring Your Own Vulnerable Driver) technique to terminate security agents in kernel space. Enable Microsoft Recommended Driver Blocklists and Hypervisor-Protected Code Integrity (HVCI) via Active Directory Group Policies. Additionally, configure SIEM alerts for Sysmon Event ID 6 (Driver Loaded) pointing to known vulnerable drivers.

3. **Enforce Dependency Pinning and Signature Verification**
   Supply chain compromises (such as Sapphire Sleet targeting Mastra AI packages) exploit floating tags in action files and unpinned public package registries. Configure development environments to pin all npm, PyPI, and GitHub dependencies to specific, immutable commit hashes (SHA-256) rather than version tags. Run npm installs with the '--ignore-scripts' flag on all production and CI/CD runners.

## Recommandations Stratégiques

1. **Imposer le MFA matériel FIDO2 sur le périmètre**
   Le MFA traditionnel par SMS ou push est régulièrement contourné par ingénierie sociale de helpdesks (Scattered Spider) ou réutilisation d'identifiants (FortiBleed). Déployez des clés matérielles FIDO2/WebAuthn (YubiKey 5, Google Titan) comme seul facteur d'administration pour les pare-feux, serveurs VPN et comptes AD privilégiés. Révoquez toutes les sessions OIDC/OAuth actives dès le déploiement.

2. **Bloquer la virtualisation non autorisée via des règles AppLocker**
   Des groupes comme 'The Gentlemen' exécutent des agents de chiffrement Linux au sein de machines virtuelles QEMU locales pour échapper à la détection de l'EDR Windows. Implémentez des règles AppLocker ou WDAC restrictives pour interdire le lancement d'exécutables de virtualisation (tels que qemu-system-x86_64.exe ou VirtualBox.exe) sur l'ensemble des postes de travail non autorisés.

## Recommandations Opérationnelles

1. **Changer les mots de passe FortiGate et désactiver l'accès WAN**
   Le dataset FortiBleed contient des identifiants d'administration compromis pour plus de 73 000 pare-feux. Auditez immédiatement vos appliances FortiGate et Sophos, renouvelez l'ensemble des mots de passe d'administration et configurez des règles de filtrage pour bloquer le management depuis le WAN. Restreignez l'accès à un VLAN d'administration isolé accessible via un bastion de sécurité.

2. **Activer le blocage des pilotes vulnérables via GPO**
   Les utilitaires de neutralisation comme GentleKiller exploitent la technique BYOVD pour arrêter les agents EDR dans l'espace noyau. Activez la liste de blocage des pilotes recommandée par Microsoft et l'intégrité du code protégée par hyperviseur (HVCI) via GPO. Configurez des alertes de détection pour l'Event ID 6 de Sysmon (Driver Loaded) signalant des pilotes anormaux.

3. **Imposer le verrouillage SHA des dépendances logicielles**
   Les attaques de supply chain (telle que Sapphire Sleet sur Mastra AI) exploitent les dépôts de paquets non verrouillés pour injecter du code lors du build. Forcez les équipes de développement à lier l'ensemble des dépendances à des hashs de commits SHA (SHA-256) au lieu de simples tags de version dans package.json, et utilisez l'argument '--ignore-scripts' lors de builds.
