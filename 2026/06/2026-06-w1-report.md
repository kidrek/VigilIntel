## Trends & Weak Signals

**Emerging techniques:**
- Abusing legitimate web app builders like Typedream for evasive phishing hosting
- Cookie forgery attacks targeting edge VPN gateways to bypass MFA completely
- Prompt injection attacks and privilege escalation on enterprise AI agent systems

**Declining techniques:**
- Standard SMS and basic TOTP MFA due to the rapid rise of session hijacking and cookie theft tools
- Unmonitored, direct outgoing developer access to public package registries like NPM and PyPI

**Geopolitical context:** The coordinated targeting of edge networking devices and SOHO routers in the Asia-Pacific region indicates a strategic push by nation-state and highly organized actors to build persistent botnet infrastructure. This network is designed to route malicious traffic and establish covert access to remote employees working for critical defense and commercial targets.

## Strategic Recommendations

1. **Enforce FIDO2 Hardware MFA for Developer Accounts**
   Traditional SMS or TOTP MFA is routinely bypassed by session hijacking and infostealers observed this period. Deploy FIDO2/WebAuthn hardware security keys (e.g., YubiKey 5) across all developer, administrator, and high-privilege sessions to eliminate phishing and session theft. This directly mitigates the risk of identity compromise seen in recent credential theft campaigns.

2. **Decommission Legacy SOHO Edge Devices Immediately**
   Multiple critical buffer overflows (CVE-2026-10187, CVE-2026-10183) are actively being targeted in Asia-Pacific to build botnet architectures. Audit all remote worker networks and replace EOL devices like TRENDnet TEW-432BRP with centrally managed, secure routers. This action prevents attackers from establishing a persistent foothold on remote corporate entry points.

## Operational Recommendations

1. **Apply Urgent PAN-OS Hotfixes for CVE-2026-0257**
   The critical GlobalProtect vulnerability CVE-2026-0257 is under active exploitation for session cookie forgery. Immediately upgrade all PAN-OS firewall clusters to the latest patched versions recommended by Palo Alto Networks. Configure the firewalls to reject shared certificate configurations for session encryption to prevent cookie replication.

2. **Audit Local Admin Accounts via EDR Queries**
   Threat actors like Stormous leverage compromised local accounts to move laterally and execute double-extrusion ransomware. Run the EDR query 'DeviceProcessEvents | where ProcessCommandLine contains "net user"' daily to detect unauthorized credential creation. Isolate any host immediately if anomalous local accounts are detected.

## Recommandations Stratégiques

1. **Exiger le MFA Matériel FIDO2 pour les Développeurs**
   Le MFA SMS ou TOTP traditionnel est systématiquement contourné par le vol de session et les infostealers observés cette période. Déployez des clés de sécurité matérielles FIDO2/WebAuthn (ex: YubiKey 5) sur tous les comptes d'administration et de développement pour éliminer le vol de session. Cela atténue directement le risque de compromission d'identité constaté récemment.

2. **Mettre hors service les routeurs SOHO obsolètes**
   Plusieurs débordements de tampon critiques (CVE-2026-10187, CVE-2026-10183) sont activement ciblés en Asie-Pacifique pour créer des botnets. Auditez les réseaux des télétravailleurs et remplacez les routeurs en fin de vie par des équipements sécurisés et gérés de manière centralisée. Cette mesure empêche les attaquants d'établir un point d'accès persistant.

## Recommandations Opérationnelles

1. **Appliquer les correctifs PAN-OS pour la CVE-2026-0257**
   La vulnérabilité critique GlobalProtect CVE-2026-0257 fait l'objet d'une exploitation active pour la contrefaçon de cookies. Mettez immédiatement à niveau tous les clusters de pare-feu PAN-OS vers les dernières versions corrigées par Palo Alto. Configurez les pare-feu pour rejeter les certificats partagés afin d'empêcher la réplication des cookies.

2. **Auditer les comptes d'administration locaux via EDR**
   Les attaquants comme Stormous exploitent des comptes locaux compromis pour se déplacer latéralement et déployer des ransomwares. Exécutez quotidiennement la requête EDR 'DeviceProcessEvents | where ProcessCommandLine contains "net user"' pour détecter toute création de compte non autorisée. Isolez immédiatement tout hôte suspect.
