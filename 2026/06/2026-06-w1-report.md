## Trends & Weak Signals

**Emerging techniques:**
- Bring Your Own Hypervisor: deploying lightweight local QEMU virtual machines to encrypt host systems outside EDR visibility.
- Invisible Unicode Obfuscation: leveraging zero-width Unicode characters inside comments of platforms like Steam as Dead Drop Resolvers.
- AI-assisted Account Hijacking: prompt injection techniques targeting customer-support chatbots to bypass identity verification.
- Delayed Tool Invocation: using indirect prompt injection via mobile notifications to hijack system assistant toolbars.

**Declining techniques:**
- Traditional SMS or TOTP-based Multi-Factor Authentication (highly vulnerable to prompt-injection and proxy-based session hijacking).
- Cleartext script execution via standard cmd.exe paths (instantly blocked by modern behavior-based EDR engines).

**Geopolitical context:** Geopolitical cyber operations have escalated into hybrid formats. Iran is merging cyber-espionnage, hack-and-leak campaigns under the 'Handala' persona, and physical sabotage recruitment within Western countries. Meanwhile, the South China Sea remain areas of intense cyber-espionage by Chinese state actors mapping maritime logistics, and European defense entities are increasing sovereign space-based communications to counter Russian intercept campaigns.

## Strategic Recommendations

1. **Enforce FIDO2 Hardware MFA Everywhere**
   Standard MFA (SMS, TOTP, push notifications) is routinely bypassed by modern session hijacking and AI-driven support bypasses (observed in Meta/Instagram hijacking this period). Mandate FIDO2/WebAuthn-compliant hardware security keys (e.g., YubiKey 5, Google Titan) for all enterprise directory, development, and cloud administrative accounts. Revoke active sessions immediately upon rolling out these hardware-backed controls.

2. **Implement Strict Software Dependency Control**
   Supply chain compromise via npm/PyPI (the Miasma worm by TeamPCP) exploits automated pipelines and floating tags. Transition code development frameworks to enforce commit-hash pinning (SHA-256) instead of tag names for all external resources. Mandate local dependency caching through a lightweight security proxy to scan package integrity before integration.

## Operational Recommendations

1. **Restrict local hypervisor and RMM execution**
   Operator groups like Payouts King deploy QEMU to encrypt files outside EDR visibility, and UNC3753 leverages unauthorized RMM tools (AnyDesk, Zoho). Deploy an AppLocker or WDAC policy blocking the execution of virtualization binaries (qemu-system-x86_64.exe, qemu-img.exe) on all servers not explicitly designated as hypervisors. Apply the same block rules for unapproved RMM tools.

2. **Apply Cisco Catalyst SD-WAN Patches**
   Attackers are actively exploiting CVE-2026-20245 in the wild to execute root-level commands on Catalyst SD-WAN Manager. Immediately isolate the SD-WAN administrative portal to an Out-of-Band (OOB) network segment. Run the 'request admin-tech' command to archive and audit system logs for command injection artifacts, and apply the manufacturer's cumulative patch.

## Recommandations Stratégiques

1. **Imposer le MFA matériel FIDO2 systématiquement**
   Le MFA traditionnel (SMS, TOTP, push) est régulièrement contourné par le vol de session et la manipulation d'IA d'assistance (campagnes Meta/Instagram). Déployez de manière obligatoire des clés de sécurité matérielles conformes FIDO2 (YubiKey 5, Google Titan) pour tous les accès d'administration cloud et de développement. Révoquez toutes les sessions actives lors du déploiement.

2. **Sécuriser rigoureusement les dépendances logicielles**
   Les attaques supply-chain npm/PyPI (ver Miasma de TeamPCP) exploitent les pipelines de build et les balises flottantes. Imposez le verrouillage des dépendances par empreintes cryptographiques (SHA-256) plutôt que par tags de versions. Déployez un proxy de cache local sécurisé pour analyser la composition des paquets avant toute compilation.

Isolez immédiatement l'accès d'administration vers un segment réseau hors bande (OOB). Utilisez la commande 'request admin-tech' pour collecter et auditer les traces système avant d'installer le correctif cumulatif d'urgence.
