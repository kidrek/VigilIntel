## Executive Summary

Week 1 of June 2026 highlights a significant shift in attacker tactics, dominated by the exploitation of critical network edge infrastructure, advanced supply chain poisoning, and hypervisor-level security evasion. Highly sophisticated actors (including APT29, APT41, and Void Manticore) are actively abusing perimeter devices, while cybercriminals deploy malware like JS.MonoGlyphRAT, Lumma Stealer, and Sectop RAT via complex infection chains. ## Trends & Weak Signals

**Emerging techniques:**
- Bring Your Own Hypervisor (BYOH) using nested QEMU installations to run ransomware outside EDR scope
- ECMAScript payloads embedded inside native Scalable Vector Graphic (SVG) files to bypass email sandbox engines
- Zero-width invisible Unicode character obfuscation in public Steam profiles used as Dead Drop Resolvers (DDR)

**Declining techniques:**
- Direct un-obfuscated script executions (such as plain powershell.exe calls) on endpoints
- Basic HTTP/cleartext C2 communications without SSL/TLS wrapping or non-standard custom encoding layers

**Geopolitical context:** Geopolitical friction in the South China Sea acts as a catalyst for cyber campaigns by Chinese actors (APT41) mapping logistics networks, while the ongoing Ukraine-Russia conflict drives Gamaredon's continuous deployment of administrative and military data exfiltration routines. In parallel, Middle-Eastern tensions trigger destructive cyber-physical hybrid espionage models orchestrated by the Iranian MOIS under the Handala persona.

## Strategic Recommendations

1. **Enforce FIDO2 Hardware MFA on Cloud Portals**
   Unsecured cloud environments are being systematically breached via credential stuffing and session hijacking (as seen in Snowflake attacks on Neiman Marcus). Organizations must mandate FIDO2 WebAuthn hardware security keys (such as YubiKey 5 Series or Google Titan) for all administrative and SaaS access vectors. Implementing this architecture invalidates stolen credentials, mitigating the threat of Lumma and other modern infostealers.

2. **Block Unauthorized Local Hypervisors Globally**
   Ransomware groups like Payouts King are deploying nested virtual machines using the legitimate QEMU hypervisor to execute encryption pipelines outside host EDR/EPP visibility. Implement strict Application Control policies using Windows Defender Application Control (WDAC) to block unauthorized execution of virtualization binaries (specifically `qemu-system-x86_64.exe` and `qemu-img.exe`). This neutralizes BYOH defense evasion techniques on production servers.

## Operational Recommendations

1. **Deactivate Default Windows Script Host Execution**
   The JS.MonoGlyphRAT campaign and similar vectors rely on Windows Script Host (`wscript.exe`) to execute malicious JavaScript attachments from standard user locations. Operational teams must immediately run the registry command `reg add HKLM\Software\Microsoft\Windows Script Host\Settings /v Enabled /t REG_DWORD /d 0 /f` to globally disable WSH on all non-developer endpoints. This prevents execution of `.js`, `.vbs`, and `.wsf` script files.

2. **Disable npm Package Postinstall Script Execution**
   Supply chain poisoning operations (such as TeamPCP's compromise of Red Hat's npm repository) rely on automated postinstall script executions to deploy payloads like the Mini Shai-Hulud worm. Configure all local developer workstations and CI/CD build environments to run npm with the `--ignore-scripts` parameter. Ensure this directive is locked globally via `.npmrc` configuration files.

## Secteurs & Géographies Ciblés

**Secteurs (classification) :** Infrastructures critiques, Gouvernement et Défense, Finance, Santé et Sciences de la vie, Technologie et Télécom, Industrie et Fabrication, Commerce de détail et Consommation, Non lucratif et ONG

**Pays :** États-Unis, Japon, Inde, Palestine, Mexique, Ukraine, France

**Panorama détaillé :**

- **Gouvernement et Défense :** La municipalité d'Urasoe (Japon) a subi le vol physique de 83 ordinateurs de sous-traitants exposant les PII de 115 526 résidents. Des ambassades européennes ont été ciblées par le groupe étatique russe APT29 (Cozy Bear) via des leurres d'invitations diplomatiques visant à installer des backdoors "Wipry".
- **Technologie et Télécom :** Plus de 30 packages logiciels npm hébergés par Red Hat sous l'espace de noms `@redhat-cloud-services` ont été infectés par TeamPCP, distribuant le ver "Mini Shai-Hulud" pour dérober des jetons GitHub et des clés AWS de développeurs.
- **Commerce de détail et Consommation :** Le détaillant de luxe Neiman Marcus a informé 64 000 clients d'une violation de base de données compromettant des noms, des historiques de transactions et des numéros de carte partiels suite à l'exploitation d'un portail Snowflake dépourvu de MFA.
- **Non lucratif et ONG :** La base de données du Programme Alimentaire Mondial (PAM) de l'ONU, contenant les allocations d'aide et les profils d'identité sensibles de 600 000 ménages dans la région de Gaza, a été divulguguée, posant des risques géopolitiques majeurs.
- **Santé et Sciences de la vie :** Les prestataires de santé CS Insurance (Mexique) et Ace Hospital (Inde) ont vu leurs dossiers cliniques, systèmes de facturation et dossiers médicaux de patients compromis et publiés sur le dark web par le groupe de rançongiciel KillSec3.
- **Industrie et Fabrication :** Le fabricant japonais Koa Glass a été contraint à un arrêt opérationnel partiel suite à une campagne de chiffrement ciblée menée par le nouveau groupe de rançongiciel 'The Gentlemen'.
- **Infrastructures critiques :** Des acteurs étatiques, notamment le groupe chinois APT41 et le groupe russe Gamaredon, ont ciblé de manière persistante des réseaux logistiques et des hubs de transport, y compris des opérations maritimes critiques en mer de Chine méridionale, pour des collectes de données en mémoire.
- **Finance :** Le géant financier HDFC (Inde) a été confronté à l'exfiltration massive de dossiers d'investisseurs confidentiels, entraînant une injonction d'urgence de la Haute Cour de Bombay pour interdire la diffusion des données volées.

## Tendances & Signaux Faibles

**Techniques émergentes :**
- Apportez votre propre hyperviseur (BYOH) avec installation de QEMU pour exécuter des ransomwares hors de portée des EDR
- Charges utiles ECMAScript intégrées dans des fichiers d'images vectorielles SVG pour contourner les moteurs de messagerie
- Obfuscation de caractères Unicode invisibles de largeur nulle dans des profils Steam publics utilisés comme serveurs de rebond (DDR)

**Techniques en déclin :**
- Exécutions directes de scripts non obfusqués (telles que des appels powershell.exe simples) sur les terminaux
- Communications C2 HTTP en clair de base sans encapsulation SSL/TLS ou couches de codage personnalisées non standard

**Contexte géopolitique :** Les frictions géopolitiques en mer de Chine méridionale agissent comme un catalyseur pour les cybercampagnes d'acteurs chinois (APT41) cartographiant les réseaux logistiques, tandis que le conflit Ukraine-Russie alimente le déploiement continu par Gamaredon de routines d'exfiltration de données administratives et militaires. En parallèle, les tensions au Moyen-Orient déclenchent des modèles d'espionnage hybrides cyber-physiques destructeurs orchestrés par le MOIS iranien sous l'identité Handala.

## Recommandations Stratégiques

1. **Imposer le MFA matériel FIDO2 sur le Cloud**
   Les environnements cloud non sécurisés sont systématiquement compromis par bourrage d'identifiants et vol de session (comme observé lors des attaques Snowflake sur Neiman Marcus). Les organisations doivent imposer l'usage de clés de sécurité matérielles FIDO2 WebAuthn (telles que la série YubiKey 5 ou Google Titan) pour tous les accès administratifs et SaaS. L'adoption de cette architecture invalide les identifiants volés par Lumma ou d'autres infostealers.

2. **Bloquer globalement les hyperviseurs locaux non autorisés**
   Des groupes de rançongiciels comme Payouts King déploient des machines virtuelles imbriquées à l'aide de l'hyperviseur légitime QEMU pour chiffrer les disques hors de portée des EDR/EPP hôtes. Mettez en œuvre des règles strictes de contrôle d'applications via Windows Defender Application Control (WDAC) pour interdire l'exécution de binaires de virtualisation non autorisés (notamment `qemu-system-x86_64.exe` et `qemu-img.exe`). Cela neutralise les techniques d'évasion BYOH sur les serveurs de production.

## Recommandations Opérationnelles

1. **Désactiver l'exécution par défaut de Windows Script Host**
   La campagne JS.MonoGlyphRAT et d'autres vecteurs similaires s'appuient sur Windows Script Host (`wscript.exe`) pour exécuter des scripts JavaScript malveillants à partir de répertoires utilisateur. Les équipes opérationnelles doivent exécuter immédiatement la commande de registre `reg add HKLM\Software\Microsoft\Windows Script Host\Settings /v Enabled /t REG_DWORD /d 0 /f` pour désactiver WSH sur tous les postes de travail non développeurs. Cela bloque l'exécution des fichiers `.js`, `.vbs` et `.wsf`.

2. **Désactiver l'exécution des scripts de post-installation npm**
   Les attaques de supply chain (telles que la compromission du dépôt npm de Red Hat par TeamPCP) s'appuient sur l'exécution de scripts post-installation automatisés pour déployer le ver Mini Shai-Hulud. Configurez tous les postes de développeurs et les environnements de build CI/CD pour exécuter npm avec le paramètre `--ignore-scripts`. Verrouillez cette consigne de manière globale via les fichiers de configuration `.npmrc`.
